📌 右优先原则

中缀转前缀时，使用右优先原则： 当运算符优先级相同时，优先让右边的运算符先生效。

算法：从右向左扫描，遇到右括号入栈，左括号弹出到右括号。

📚 三种算术表达式

中缀表达式

运算符在操作数中间

a + b

人类最熟悉的形式，需要括号确定优先级

后缀表达式（逆波兰）

运算符在操作数后面

a b +

不需要括号，计算机处理效率高

前缀表达式（波兰）

运算符在操作数前面

+ a b

同样不需要括号，应用较少

🔄 转换规则

中缀 → 后缀（左优先）

  1. 1. 从左到右扫描中缀表达式
  2. 2. 遇到操作数：直接输出
  3. 3. 遇到运算符：
    • • 栈空或栈顶是 ( ：直接入栈
    • • 否则：弹出所有优先级 ≥ 当前的运算符，再入栈
  4. 4. 遇到 ( ：直接入栈
  5. 5. 遇到 ) ：弹出直到遇到 (
  6. 6. 扫描完：弹出栈中所有运算符

中缀 → 前缀（右优先）

  1. 1. 从右到左扫描中缀表达式
  2. 2. 遇到操作数：直接输出
  3. 3. 遇到运算符：
    • • 栈空或栈顶是 ) ：直接入栈
    • • 否则：弹出所有优先级 > 当前的运算符
  4. 4. 遇到 ) ：直接入栈
  5. 5. 遇到 ( ：弹出直到遇到 )
  6. 6. 扫描完：弹出栈中所有运算符，最后逆序输出

🧮 求值规则

后缀表达式求值

  1. 1. 从左到右扫描
  2. 2. 遇到操作数：入栈
  3. 3. 遇到运算符：
    • • 弹出两个操作数
    • • 先弹出的是右操作数
    • • 计算结果入栈
  4. 4. 最后栈中唯一元素即结果

前缀表达式求值

  1. 1. 从右到左扫描
  2. 2. 遇到操作数：入栈
  3. 3. 遇到运算符：
    • • 弹出两个操作数
    • • 先弹出的是左操作数
    • • 计算结果入栈
  4. 4. 最后栈中唯一元素即结果

⚠️ 易错点对比

后缀求值

先弹出 = 右操作数

a b - → a - b

前缀求值

先弹出 = 左操作数

- a b → a - b
