# 循环队列 (顺序存储)

> [!summary] 核心考点速览
> 1.  **取模运算**：实现逻辑上的环状结构。
> 2.  **判空/判满**：是本节最核心的难点，有三种方法（牺牲空间、size变量、tag变量）。
> 3.  **指针定义**：`rear` 是指向**下一个位置**（主流）还是**当前位置**（非主流但在考）。
> 4.  **队长公式**：必考计算。

## 1. 核心机制：取模 (Modulo)

为了解决普通顺序队列的“假溢出”问题（前面有空位但`rear`已达上限），将存储空间逻辑上变为环状。

*   **下标索引变化**：`index = (index + 1) % MaxSize`
*   **物理结构**：静态数组 `data[MaxSize]`。
*   **逻辑结构**：环状队列。

## 2. 场景一：Rear指向队尾元素的下一个位置（最常考）

**设定**：
*   `front`：指向队头元素。
*   `rear`：指向队尾元素的**后一个**位置（即下一个插入位）。
*   **初始化**：`front = rear = 0`。

### 2.1 判空与判满的三种策略

此场景下，`front == rear` 既可能为空，也可能为满，需引入额外手段区分。

| 策略 | 判空条件 | 判满条件 | 牺牲空间? | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **A. 牺牲一个单元** <br> (默认/最推荐) | `front == rear` | `(rear + 1) % MaxSize == front` | **是** | **考研默认方式**。队列最多存 `MaxSize - 1` 个元素。 |
| **B. 增加 size 变量** | `size == 0` | `size == MaxSize` | 否 | 每次操作需维护 `size++` 或 `size--`。 |
| **C. 增加 tag 变量** | `front == rear && tag == 0` | `front == rear && tag == 1` | 否 | `tag=0`表示最近一次是删除；`tag=1`表示最近一次是插入。 |

### 2.2 关键操作代码逻辑 (以策略A为例)

*   **入队 (Enqueue)**：
    1.  判满：`if ((rear + 1) % MaxSize == front) return false;`
    2.  存值：`data[rear] = x;`
    3.  移动：`rear = (rear + 1) % MaxSize;`
*   **出队 (Dequeue)**：
    1.  判空：`if (rear == front) return false;`
    2.  取值：`x = data[front];`
    3.  移动：`front = (front + 1) % MaxSize;`
*   **计算队长 (Length)**：
> [!tip] 万能公式 (背诵)
> Q[0 … MaxSize-1]
> `Len = (rear - front + MaxSize) % MaxSize`
> *原理：`(rear - front)` 可能是负数，加上 `MaxSize` 修正为正，再取模。*

## 3. 场景二：Rear指向队尾当前元素（避坑指南）

**设定**：
*   `front`：指向队头元素。
*   `rear`：指向**当前**队尾元素。
*   **初始化**：`front = 0`, `rear = n - 1` (或题目特殊指定)。

### 3.1 核心差异

> [!danger] 极其容易丢分点
> 1.  **入队操作顺序**：**先移指针，后存数据**。
>     *   `rear = (rear + 1) % MaxSize;`
>     *   `data[rear] = x;`
> 2.  **判空/判满**：如果不引入辅助变量（size/tag），判别逻辑会变得很复杂（通常会牺牲空间：`front`前一个位置不能存，即`rear`追上`front`的前两步算满等，具体看题目约定）。
> 3.  **建议**：若题目考察此场景，通常会搭配 `size` 或 `tag` 变量考察，否则逻辑歧义较大。

## 4. 考研做题策略 (功利化总结)

在做选择题或大题时，严格按照以下步骤扫描题目（**审题流程**）：

1.  **Step 1: 确认指针定义**
    *   看 `rear` 是指**下一个可插入位置**（Next）还是**当前队尾元素**（Current）？
    *   *线索*：看初始值，若 `rear=0` 且 `front=0`，通常是指向下一个；若代码里是 `rear = (rear+1)%N; data[rear]=x;` 则是指当前。

2.  **Step 2: 确认判满方式**
    *   看是否牺牲了一个存储单元？
    *   看是否有 `size` 或 `tag` 辅助变量？

3.  **Step 3: 套用公式**
    *   若求长度：直接套 `(rear - front + MaxSize) % MaxSize` (前提是rear指向下一个)。
    *   若 rear 指向当前：长度通常是 `(rear - front + 1 + MaxSize) % MaxSize` (具体带入特值验证)。

### 易混淆对比表

| 特性 | **类型 I (主流)** | **类型 II (非主流)** |
| :--- | :--- | :--- |
| **Rear 含义** | 指向**下一个**待插入位置 | 指向**当前**已有元素 |
| **初始化** | `front=0, rear=0` | 可能 `front=0, rear=n-1` |
| **入队顺序** | 先存数据，后移指针 | **先移指针，后存数据** |
| **判满 (牺牲空间法)** | `(rear + 1) % MaxSize == front` | 较少直接考，通常结合辅助变量 |
| **元素个数** | `(rear - front + MaxSize) % MaxSize` | `(rear - front + 1 + MaxSize) % MaxSize` (需验证) |

> [!example] 验证技巧（防丢分杀手锏）
> 假如考试忘了公式或逻辑混乱：
> 1.  **画图**：画一个长度为 5 的圆环。
> 2.  **模拟**：手动插入 2 个元素。
> 3.  **代入**：看此时 `front` 和 `rear` 的值，代入选项公式看哪个等于 2。
> *不要盲目自信，手动模拟耗时 30 秒，但能保证 100% 正确。*
