# 朴素模式匹配算法 (Brute-Force)

> [!summary] 核心考点摘要
> 1.  **本质**：暴力枚举。主串中所有长度为 $m$ 的子串依次与模式串对比。
> 2.  **指针回溯**：这是最易错的考点。匹配失败时，主串指针 $i$ 回退，模式串指针 $j$ 重置。
> 3.  **时间复杂度**：最坏 $O(nm)$，用于对比 KMP 算法的优势。

## 1. 概念界定 (术语统一)
*   **主串 (S, $n$)**：被搜索的长字符串。
*   **模式串 (T, $m$)**：要查找的目标短字符串。
*   **子串 vs 模式串**：
    *   子串：主串中实际存在的片段。
    *   模式串：想找的目标，主串中**不一定**存在。
*   **目标**：找到 $T$ 在 $S$ 中的位置，返回起始下标；否则返回 0。

## 2. 算法执行逻辑 (可视化)

设定两个指针：$i$ 指向主串，$j$ 指向模式串。若数组下标从 1 开始：

### 情况 A：字符匹配
$$S[i] == T[j]$$
*   **操作**：`i++`, `j++` (继续比对下一位)

### 情况 B：字符失配 (关键!)
$$S[i] \neq T[j]$$
*   **逻辑**：当前子串匹配失败，主串指针需回到**下一个子串的起始位置**，模式串复位。
*   **指针回溯公式**：
    *   $j \leftarrow 1$ (模式串重头开始)
    *   $i \leftarrow i - j + 2$
*   **公式推导 (如果不理解，死记)**：
    *   当前匹配的子串起始位置是 $i - j + 1$。
    *   下一个子串起始位置自然是 $(i - j + 1) + 1 = i - j + 2$。

### 演示图解
```text
Step 1: 匹配中
S:  a  b  a  b  c  d ...
    ^ i
T:  a  b  c
    ^ j
(a=a, b=b, 此时 i,j 同步后移)


Step 2: 失配时刻
S:  a  b  a  b  c  d ...
          ^ i (此时指向 'a')
T:  a  b  c
          ^ j (此时指向 'c')
(a!=c, 匹配失败)


Step 3: 回溯 (i = i-j+2, j=1)
S:  a  b  a  b  c  d ...
       ^ i (回退到第二个字符)
T:     a  b  c
       ^ j (重置)
```

## 3. 代码模板 (王道/408标准)

> [!TIP] 拿分技巧
> 考研代码题通常默认下标从 **1** 开始 (S[0] 存放长度或废弃)。若题目规定从 0 开始，逻辑微调即可，但核心回溯思想不变。

```c
int Index(SString S, SString T) {
    int i = 1, j = 1;
    while (i <= S.length && j <= T.length) {
        if (S.ch[i] == T.ch[j]) {
            // 匹配成功，继续后续字符
            ++i; 
            ++j; 
        } else {
            // 匹配失败，指针回溯
            i = i - j + 2; // 核心得分点
            j = 1;
        }
    }
    // j超出T长度，说明匹配成功
    if (j > T.length) 
        return i - T.length;
    else 
        return 0;
}
```

## 4. 时间复杂度分析 (选择题考点)

*   **匹配次数**：最多匹配 $n-m+1$ 个子串。
*   **最好情况**：$O(1)$ 或 $O(n+m)$ (视具体教材定义，一般不考)。
*   **最坏情况**：**$O(nm)$**
    *   **场景**：每次都在最后一个字符失配。
    *   **例子**：
        *   $S$: `aaaaaab`
        *   $T$: `aaab`
        *   每次都要比对 $m$ 次才发现不行，一共比对约 $n$ 次。
    *   **推导**：$(n-m+1) \times m \approx O(nm)$ (当 $n \gg m$)。

## 5. 必背结论 (不丢分自查)

1.  **回溯是低效的根源**：朴素算法慢就慢在 $i$ 指针要回溯 (走回头路)。
2.  **KMP的引出**：为了解决 $i$ 回溯的问题，引入 KMP 算法，使 $i$ 不回溯，仅 $j$ 变动。
3.  **下标计算**：如果题目给的是数组下标 **0** 开始：
    *   $i$ 回溯为：`i = i - j + 1`
    *   $j$ 回溯为：`j = 0`
    *   返回位置：`i - j` 或 `i - T.length`
