> [!danger] 功利化导读：为什么学？
> 统考大纲**无明确要求**掌握该算法本身，但它的**核心思想（空间换时间、哈希映射统计）**是历年算法大题（Q41）拿满分的核心杀器。
> **目标985的底层逻辑：** 当题目要求“时间尽可能高效”（暗示不在乎空间）或发现元素取值范围 $k$ 极小（$k$ 与 $n$ 同数量级）时，立刻条件反射使用**计数排序思想**，保底 $O(n)$ 满分时间复杂度！

### 一、 核心算法思想与步骤推演

**本质：** 通过辅助数组的索引来映射待排序元素的值，统计各个元素的出现次数，进而确定最终位置。
**前提约束：** 待排序元素须为整数，且取值范围 $k$ 不能过大。

**满分推演三步曲（必须在草稿纸上能盲写）：**
假设待排序数组 `A` 范围为 $[0, k]$，长度为 $n$。辅助数组为 `C[k+1]`，结果数组为 `B[n]`。

1. **频率统计（Step 1）：** 
   遍历 `A`，遇到 `A[i] = x`，则 `C[x]++`。
   *意义：`C[x]` 此时表示值为 $x$ 的元素共有多少个。*
2. **累加求和（Step 2）：**
   对 `C` 从前向后做前缀和：`C[i] = C[i] + C[i-1]`。
   *意义：`C[i]` 此时表示 **小于等于 $i$ 的元素累计有多少个**。即指明了元素 $i$ 在排序后数组的最后位置。*
3. **倒序回填（Step 3）—— 保证不丢分的关键：**
   **从后往前**（$i$ 从 $n-1$ 到 $0$）遍历原数组 `A`：
   - 查 `C` 数组：`C[A[i]]` 的值减 1（即 `C[A[i]]--`），得到的位置即为 `A[i]` 在结果数组 `B` 中的真实下标。
   - 填入 `B` 数组：`B[ C[A[i]] ] = A[i]`。
   *考点：为什么必须从后往前？为了保证**稳定性（Stability）**，后出现的相等元素会被放在更靠后的位置。*

### 二、 性能剖析（答题卡上复杂度分析怎么写）

想要一分不丢，算法题第二问的复杂度必须精确。设元素个数为 $n$，最大与最小值之差为 $k$。

*   **时间复杂度：** $O(n + k)$
    *   *解析：* 找最值遍历 $O(n)$，初始化辅助数组 $O(k)$，计数和回填 $O(n)$。
    *   *对比界限：* 当 $k = O(n)$ 时，退化为 $O(n)$，吊打快排和堆排；若 $k > n\log_2 n$，则放弃，老实回去用快排。
*   **空间复杂度：** $O(n + k)$
    *   *解析：* 需要结果数组 $O(n)$ 和计数数组 $O(k)$。典型的**空间换时间**。
*   **稳定性：** **稳定**。（倒序遍历分配保证了相对顺序）。

### 三、 满分真题实战套路 (Q41 降维打击)

碰到以下两种题干描述，直接套用计数排序思想，直接奔着15分满分去：

#### 场景 1：已知取值范围极小 (2013年 Q41 主元素问题)
*   **题干特征：** 给定长度为 $n$ 的整数数组，元素取值范围在 $0$ 到 $n-1$ 之间。找出现次数大于 $n/2$ 的主元素。
*   **破题秒杀：** 元素取值最大不过 $n-1$，即 $k < n$。直接开辟一个大小为 $n$ 的辅助计数数组 `C`，遍历一次原数组统计次数，再遍历一次 `C` 找有没有大于 $n/2$ 的。
*   **得分点：** 时间 $O(n)$，空间 $O(n)$，纯暴力但拿满分！

#### 场景 2：找缺失的数字 (2018年 Q41 未出现的最小正整数)
*   **题干特征：** 找数组中未出现的最小正整数，要求“时间上尽可能高效”（暗示空间随便用）。含有负数和任意大数。
*   **破题秒杀（抽屉原理 / 鸽巢原理）：** 
    数组长度为 $n$，最极端情况是塞满了 $1, 2, 3 \dots n$，那么缺失的最小正整数就是 $n+1$。因此，答案**必定**落在 $[1, n+1]$ 这个区间内！
*   **满分算法逻辑：**
    1. 申请一个长度为 $n+2$ 的辅助数组 `C`，初始化全为 0。
    2. 遍历原数组，只要遇到的数 $x$ 满足 $0 < x \le n$，就把 `C[x]` 标记为 1。（无视负数和大于 $n$ 的数，因为它们不影响最终结果）。
    3. 从左到右扫描 `C[1]` 到 `C[n+1]`，遇到的第一个值为 0 的下标 $i$，就是缺失的最小正整数。
*   **得分点：** 时间 $O(n)$，空间 $O(n)$。不排序，只计数，思维降维打击。

### 四、 考场通用健壮代码复现 (C语言)

> [!tip] 阅卷老师看什么？
> 考试手写代码如果不确定 $k$ 范围，务必先找 `max` 和 `min`。辅助数组大小开 `max - min + 1`，并在操作时将下标偏移量设为 `A[i] - min`。这样既能处理负数，又能节省空间，阅卷老师一看就是基本功扎实。

```c
#include <stdio.h>
#include <stdlib.h>

// 考场直接默写这个增强版，杜绝负数越界扣分
void counting_sort_generic(int A[], int n) {
    if (n <= 1) return;

    // 1. 找最值，确定确切的 k 范围 (防止被出题人阴)
    int max = A[0], min = A[0];
    for (int i = 1; i < n; i++) {
        if (A[i] > max) max = A[i];
        if (A[i] < min) min = A[i];
    }

    // 2. 创建并初始化计数数组 (长度为 max - min + 1)
    int k = max - min + 1;
    int *C = (int *)calloc(k, sizeof(int)); // calloc自动初始化为0
    int *B = (int *)malloc(n * sizeof(int)); // 结果暂存数组

    // Step 1: 计数 (注意索引偏移：A[i] - min)
    for (int i = 0; i < n; i++) {
        C[A[i] - min]++;
    }

    // Step 2: 累加前缀和
    for (int i = 1; i < k; i++) {
        C[i] = C[i] + C[i - 1];
    }

    // Step 3: 倒序回填，保证稳定
    for (int i = n - 1; i >= 0; i--) {
        int index_in_C = A[i] - min;
        B[C[index_in_C] - 1] = A[i];
        C[index_in_C]--;
    }

    // 4. 将排序结果拷贝回原数组
    for (int i = 0; i < n; i++) {
        A[i] = B[i];
    }

    // 释放辅助空间（极其规范，不给扣分借口）
    free(C);
    free(B);
}
```
