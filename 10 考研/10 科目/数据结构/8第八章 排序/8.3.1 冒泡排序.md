> [!abstract] 985考研定海神针：冒泡排序核心宏观认知
> **大类**：交换排序（两两比较，逆序则交换）。
> **核心思想**：每趟不断将相邻元素两两对比，**一趟结束后必有一个元素（最大或最小）落入最终位置**。
> **适用结构**：既适用于**顺序表**，也完全适用于**链表**（考研常考选择题选项）。

### 一、 核心代码复现与考点映射
代码逻辑是解大题和选择题的根基，必须肌肉记忆。以下为**带“提前结束”优化**的标准考研版代码：

```cpp
// 交换函数（注意：1次swap = 3次元素移动）
void swap(int &a, int &b) {
    int temp = a;
    a = b;
    b = temp;
}

void BubbleSort(int A[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool flag = false; // 考点：本趟是否发生交换的标志
        // 从后往前冒泡（将最小元素冒到最前）
        for (int j = n - 1; j > i; j--) {
            if (A[j - 1] > A[j]) { // 考点：只有严格大于才交换，保证稳定性
                swap(A[j - 1], A[j]);
                flag = true;
            }
        }
        if (flag == false) 
            return; // 考点：若本趟0交换，说明已有序，算法提前结束
    }
}
```

### 二、 性能剖析（绝不丢分：数据必须刻在脑子里）

> [!danger] 命题陷阱警告：**比较次数** vs **交换次数** vs **移动次数**
> 考研真题极爱在“交换”与“移动”的字眼上挖坑，务必看清题干！执行1次交换(Swap) = 发生3次移动(Move)。

| 维度 | 最好情况 (初始正序) | 最坏情况 (初始逆序) | 平均情况 |
| :--- | :--- | :--- | :--- |
| **时间复杂度** | $O(n)$ | $O(n^2)$ | $O(n^2)$ |
| **空间复杂度** | $O(1)$ (仅需temp辅助变量) | $O(1)$ | $O(1)$ |
| **比较次数** | $n-1$ 次 | $\frac{n(n-1)}{2}$ 次 | - |
| **交换次数** | $0$ 次 | $\frac{n(n-1)}{2}$ 次 | - |
| **移动次数** | $0$ 次 | **$3 \times \frac{n(n-1)}{2}$ 次** | - |
| **排序趟数** | 仅 $1$ 趟 | $n-1$ 趟 | - |

### 三、 985专属·高频命题技巧总结

#### 1. 算法稳定性分析（稳抓选择题）
*   **结论**：冒泡排序是**稳定**的。
*   **深层原因**：判定条件为 `A[j - 1] > A[j]`，当两个关键字相等时，不会发生交换，原有相对顺序不会被破坏。

#### 2. 中间状态推断（最常考选择题型）
*   **做题秒杀技巧**：经过 $k$ 趟冒泡排序后，**必然有至少 $k$ 个元素已经到达最终正确位置**。
*   *真题套路*：题干给出一个初始序列和一个中间序列，问这可能是哪种排序算法经过几趟的结果？
    *   *判断标准*：看序列的最前端（或最后端）是否出现了连续的、在整个数组中极小（或极大）且有序的元素块。

#### 3. 链表的特殊适应性
*   虽然一般排序算法偏爱数组，但冒泡排序**非常适合单链表**。
*   *链表实现逻辑*：从头节点开始，扫描指针依次后移，比较相邻节点值，若逆序则交换节点内的数据域（或交换指针，但交换数据域代码更简单）。不需要随机访问特性，完美契合。

#### 4. 极限压榨边界条件
*   如果一个包含 $n$ 个元素的序列已经是完全有序的，虽然只进行 $1$ 趟排序，但这一趟依然需要进行 $n-1$ 次相邻元素的**比较**，只是不发生**交换**。不要把最好情况下的比较次数记成 $0$。
