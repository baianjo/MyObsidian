> [!abstract] 核心提分点（功利速记）
> 1. **直接插入排序**：最好 $O(n)$（正序），最坏/平均 $O(n^2)$。**稳定**。
> 2. **带哨兵写法**：`A[0]` 存当前元素，免去越界判断，常数级提速（考研代码大题推荐写法）。
> 3. **折半插入排序【易错点】**：仅减少**比较次数**，**不减少移动次数**，总时间复杂度**仍为 $O(n^2)$**。
> 4. **链表插入排序**：仅减少**移动次数**，**不减少比较次数**，总时间复杂度**仍为 $O(n^2)$**。

## 一、 直接插入排序 (Direct Insertion Sort)

**💡 算法思想**：将数组分为 `[已排序区 | 未排序区]`。每次取未排序区第一个元素，在已排序区**从后往前**找位置，比它大的统统后移，空出位置后插入。

### 1. 考研标准实现：带哨兵版本（推荐重点掌握）
实际数据从 `A[1]` 开始存，`A[0]` 留空作为**哨兵**。
*   **哨兵的作用**：暂存待插入元素；**避免在内层循环中每次都判断 `j >= 0`**，提升常数级效率。

```cpp
// 核心伪代码复现
for (int i = 2; i <= n; i++) {
    if (A[i] < A[i - 1]) {        // 仅当比前一个(已排序最大值)小才需要移动
        A[0] = A[i];              // A[0] 充当哨兵和副本
        int j;
        for (j = i - 1; A[0] < A[j]; j--) { 
            A[j + 1] = A[j];      // 大于A[0]的元素全部右移
        }
        A[j + 1] = A[0];          // 插入到正确位置
    }
}
```
*可视化演示状态*：
`[ 哨兵区 ] | [ 已排序区 ] | [ 待处理元素 ] | [ 未排序区 ]`
`[ temp  ] | 13, 38, 49 |      27      | 65, 97, 76 ` -> 38, 49 右移，27 插入。

### 2. 复杂度与稳定性（必背结论）

| 维度 | 结论 | 考点说明 |
| :--- | :--- | :--- |
| **最好时间** | **$O(n)$** | 表本身**已有序**（正序），每次只需比较 1 次，移动 0 次。 |
| **最坏/平均** | **$O(n^2)$** | 逆序（每次都要比到头并全右移）。 |
| **空间复杂度** | **$O(1)$** | 仅需常数级辅助空间（如哨兵 `A[0]` 或 `temp`）。 |
| **稳定性** | **稳定** | 相同元素不越位（代码中条件严格为 `<` 时才移动）。 |

## 二、 折半插入排序 (Binary Insertion Sort)

**💡 算法思想**：因为“已排序区”是有序的，找插入位置时没必要顺序往回找，直接上**折半查找**（二分法）。

> [!warning] 考研极高频陷阱
> 折半查找**只减少了比较次数**（降至 $O(n \log n)$），**并没有减少元素的移动次数**。
> 因此，折半插入排序的**时间复杂度依然是 $O(n^2)$**，没有质的飞跃！

### 考试必抓的 2 个代码细节（决不丢分）

1.  **如何保证“稳定性”？（选择/判断题常客）**
    *   当折半查找遇到 `A[mid] == target` 时，**不能停**！
    *   必须继续在右半区间找，即强制 `low = mid + 1`。这样能保证后出现的相同元素永远插在原元素的右边。
2.  **查找停止后的插入位置在哪？**
    *   折半停止的条件必然是 `low > high`。
    *   **最终插入位置就是 `low`**（也就是 `high + 1`）。
    *   需要将 `[low, i-1]` 区间内的所有元素统一右移一位。

```cpp
// 考研提分版折半查找核心逻辑
int low = 1, high = i - 1;
while (low <= high) {
    int mid = (low + high) / 2;
    if (A[mid] > A[0]) {
        high = mid - 1;
    } else {
        low = mid + 1; // 【重点】：即使相等也向右找，保证稳定性！
    }
}
// 循环结束后，low 所在位置即为插入位置
for (int j = i - 1; j >= low; j--) {
    A[j + 1] = A[j]; // 统一右移
}
A[low] = A[0];
```

## 三、 链表插入排序

**💡 核心对比**：数据结构换成**单链表**。
*   **找位置**：只能从头节点**顺序比较**寻找（链表不支持折半查找）。-> **比较次数仍为 $O(n^2)$**。
*   **插入动作**：只需修改指针（`next`），无需像数组那样大量移动元素。-> **移动次数降为 $O(1)$**。

> [!check] 结论总结
> 无论是折半还是换成链表，插入排序的**本质瓶颈**（找位置和腾空间的综合代价）没有打破，**总时间复杂度全部死死焊在 $O(n^2)$**。只有在序列**基本有序**时，插入排序才是神，能逼近 $O(n)$。
