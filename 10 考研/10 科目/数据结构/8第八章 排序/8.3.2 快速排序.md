## 🎯 核心考纲速记（无脑提分版）
快速排序是内部排序中**平均性能最优秀**的算法，属于**高频必考、必背手写代码**级别。
*   **时间复杂度**：最好/平均 $O(n \log_2 n)$，最坏 $O(n^2)$。
*   **空间复杂度**：最好/平均 $O(\log_2 n)$，最坏 $O(n)$（取决于递归树深度）。
*   **稳定性**：**不稳定**（如：表 `[2, 2_under, 1]` 以首元素为基准划分后变为 `[1, 2_under, 2]`，相对位置改变）。
*   **最坏情况**：初始序列**完全有序或完全逆序**时，退化为单链表，效率最差。
*   **优化方案**：三者取中法（头、中、尾取中间值作枢轴）、随机选取枢轴。

## 🧠 算法思想与执行全貌

**核心理念：分治法（Divide and Conquer）**
1.  **选取枢轴（Pivot）**：通常选当前子表的第一个元素。
2.  **一次划分（Partition）**：通过 `low` 和 `high` 双指针交替往中间扫描。
    *   `high` 找**小于**枢轴的元素，扔到左边。
    *   `low` 找**大于等于**枢轴的元素，扔到右边。
    *   当 `low == high` 时相遇，该位置即为枢轴的**最终绝对位置**。
3.  **递归处理**：对枢轴左右两侧的子表重复上述过程，直到子表长度为 0 或 1。

### 🔄 一次划分双指针运行推演（可视化脑图）
> [!example] 设序列为 `[49, 38, 65, 97, 76, 13, 27]`，枢轴 `pivot = 49`
> 1. `pivot = 49` 腾出 `low`(0) 的坑位。
> 2. `high` 左移找 $<49$ 的数 $\rightarrow$ 找到 `27`，填入 `low` 坑位。
> 3. `low` 右移找 $\ge49$ 的数 $\rightarrow$ 找到 `65`，填入 `high` 坑位。
> 4. `high` 左移找 $<49$ 的数 $\rightarrow$ 找到 `13`，填入 `low` 坑位。
> 5. `low` 右移找 $\ge49$ 的数 $\rightarrow$ 找到 `97`，填入 `high` 坑位。
> 6. `low` 与 `high` 最终在索引 `3` 处相遇，将 `49` 填入该位置。
> **结果**：`[27, 38, 13, 49(归位), 76, 97, 65]` （左边全 $<49$，右边全 $\ge49$）

## 💻 必背手写源码（极简且一分不扣）
> [!warning] 考场默写要求
> 必须肌肉记忆，注意 `while` 内部条件的 `<=` 和 `<`，以及必须要写 `low < high` 防止指针越界！

```cpp
// 1. 划分函数：确定一个元素的最终位置，返回其下标
int Partition(int A[], int low, int high) {
    int pivot = A[low]; // 选第一个元素作为枢轴，此时 low 处相当于空出了一个“坑”
    
    while (low < high) { // 大循环：只要两指针未相遇
        // high 找比枢轴小的元素
        while (low < high && A[high] >= pivot) --high; 
        A[low] = A[high]; // 填坑：把比枢轴小的元素扔到左边
        
        // low 找比枢轴大(或等于)的元素
        while (low < high && A[low] <= pivot) ++low;
        A[high] = A[low]; // 填坑：把比枢轴大的元素扔到右边
    }
    
    A[low] = pivot; // 最后相遇处 low==high，把枢轴元素放回最终位置
    return low;     // 返回枢轴的最终下标
}

// 2. 递归主函数
void QuickSort(int A[], int low, int high) {
    if (low < high) { // 递归出口：子表至少有两个元素才需要划分
        int pivotpos = Partition(A, low, high); // 划分，确定一个元素的最终位置
        QuickSort(A, low, pivotpos - 1);        // 递归排序左子表
        QuickSort(A, pivotpos + 1, high);       // 递归排序右子表
    }
}
```

## 📈 性能深度剖析（与递归树强绑定）
快速排序的时间和空间开销**完全取决于递归树的深度**。把每次划分看作二叉树的分支：

| 场景 | 状态描述 | 递归树形态 | 递归深度 | 时间复杂度 | 空间复杂度 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **最好/平均** | 每次枢轴都能**均匀平分**序列 | 平衡二叉树 | $\lfloor \log_2 n \rfloor + 1$ | $O(n \log_2 n)$ | $O(\log_2 n)$ |
| **最坏** | 初始序列**完全有序/逆序** | 单支树（退化为链表） | $n$ | $O(n^2)$ | $O(n)$ |

*(注：单次 `Partition` 需要扫描一遍子表，时间耗费为 $O(n)$，总时间 = 单层处理时间 $\times$ 递归深度)*

## 🚨 统考命题极度危险陷阱（985拉分关键）

> [!danger] “一次划分” $\neq$ “一趟排序”（真题定义与部分教材存在分歧！）
> 在统考真题的严格语境下，必须明确区分以下两个概念，否则极其容易在选择题中丢分：
> 
> 1. **一次划分（Partition）**：
>    - 对应代码中的**调用一次 `Partition` 函数**。
>    - **只确定 1 个**枢轴元素的最终位置。
>    
> 2. **一趟排序（One Pass）**：
>    - 对应递归树中的**完整的一层**（即对当前所有未确定位置的元素都扫了一遍）。
>    - **可能确定多个**元素的最终位置（例如递归到第二层时，左子表确定一个，右子表确定一个，这一趟总共确定了2个）。
>    
> **应试策略**：如果题目提到“一趟排序后”，**以真题标准为准**，即所有尚未排序的子序列都要经历一次划分操作。
