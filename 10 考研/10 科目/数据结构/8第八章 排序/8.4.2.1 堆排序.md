> [!abstract] 985上岸导语
> 堆排序是考研综合大题和选择题的**重灾区**。它融合了**完全二叉树的性质**与**选择排序的思想**。
> **拿分核心**：死记“大根堆排升序，小根堆排降序”；彻底透彻“向下调整（小元素下坠）”的比较次数与逻辑；牢记建堆时间复杂度为 $O(n)$ 而非 $O(n \log_2 n)$。

### 01. 核心属性速查表 (无死角背诵)

| 属性 | 结论 | 避坑点 / 考法 |
| :--- | :--- | :--- |
| **基础思想** | 选择排序 | 每一趟选出最大/小值加入有序序列。 |
| **排升/降序** | **大根堆 $\rightarrow$ 递增序列**<br>**小根堆 $\rightarrow$ 递减序列** | 极易搞反！大根堆每次把最大的踢到末尾，所以最后是升序。 |
| **时间复杂度** | **$O(n \log_2 n)$** | 建堆 $O(n)$ + 排序 $O(n \log_2 n)$。最好、最坏、平均全都是此值。 |
| **建堆复杂度** | **$O(n)$** | 常考选择题！建堆最多比较 $4n$ 次，不是 $O(n \log_2 n)$。 |
| **空间复杂度** | **$O(1)$** | 仅需常数级额外变量。 |
| **稳定性** | **不稳定** | 交换堆顶与堆底时，极易打乱相同元素相对顺序。 |

### 02. 物理与逻辑结构的映射 (算地址必考)
堆在逻辑上是**完全二叉树**，在物理上是**顺序存储的数组**。
*默认数组下标从 `1` 开始存储元素（`A[0]`常留作暂存用）：*

- **根节点**：`i = 1`
- **左孩子**：`2i`
- **右孩子**：`2i + 1`
- **父节点**：$\lfloor i/2 \rfloor$
- **分支节点范围**：$1 \le i \le \lfloor n/2 \rfloor$ （**建堆的起始调整位置**）
- **叶子节点范围**：$i > \lfloor n/2 \rfloor$ （叶子天然是堆，无需初始调整）

---

### 03. 核心操作一：建初始堆 (Build Heap)
**目标**：将乱序数组整理成大根堆（根 $\ge$ 左右孩子）。
**步骤**：
1. 从**最后一个分支节点**（下标为 $\lfloor n/2 \rfloor$）开始，**从后往前**依次处理。
2. 对每个节点执行**“小元素下坠”**（向下调整）：
   - 比较当前节点的左右孩子，找出**较大者**。
   - 如果当前节点 < 较大孩子，则**互换**。
   - 换下去后，可能破坏下一层的堆结构，必须**继续向下追踪对比**，直到沉底或大于等于左右孩子。

> [!warning] 考点陷阱：建堆的关键字比较次数计算
> - 如果节点有左右双亲：每下坠一层，最多需要对比 **2次** 关键字（先比较左右孩子定胜负，再比较孩子与双亲）。
> - 如果节点只有左孩子：只需对比 **1次** 关键字。
> - 建初始堆的总比较次数 $\le 4n$。

---

### 04. 核心操作二：堆排序 (Heap Sort)
**大前提**：已经建好初始堆。
**步骤**（以大根堆为例，共进行 $n-1$ 趟）：
1. **换**：将堆顶元素（最大值）与堆底元素（当前待排序序列最后一个）交换。最大值就位。
2. **断**：将刚才就位的最大值“踢出”堆的管辖范围（待排序长度 $n$ 减 1）。
3. **调**：由于此时堆顶是刚换上来的小元素，对其执行一次**“小元素下坠”**，恢复大根堆。
4. 循环直到堆中只剩 1 个元素。

> [!info] 排序过程中的比较次数
> 每趟交换后，堆顶元素最多下坠 $h-1$ 层（$h$ 为树高）。每次下坠耗时常数级，单次调整最高复杂度 $O(h) = O(\log_2 n)$。

### 05. 核心代码复现与填空预测
统考极速提分：重点盯死 `AdjustDown` 函数里的 `for` 循环推进逻辑。

```c
// 向下调整以 k 为根的子树 (小元素下坠)
void AdjustDown(int A[], int k, int len) {
    A[0] = A[k]; // A[0] 作为暂存，保存当前要下坠的元素
    
    // i 初始指向左孩子，沿值较大的子节点向下筛选
    for (int i = 2 * k; i <= len; i *= 2) {
        
        // 1. 找最大的孩子 (如果有右孩子且右孩子更大，指向右孩子)
        if (i < len && A[i] < A[i+1]) { 
            i++; 
        }
        
        // 2. 根与最大孩子对比
        if (A[0] >= A[i]) {
            break; // 根比最大孩子还大，无需下坠，直接结束
        } else {
            A[k] = A[i]; // 大孩子上浮
            k = i;       // k 更新为大孩子原位置，准备考察下一层！
        }
    }
    A[k] = A[0]; // 被调整元素放到最终位置
}

// 完整的堆排序
void HeapSort(int A[], int len) {
    // 1. 建初始堆 (从最后一个分支节点往前)
    for(int i = len / 2; i > 0; i--) {
        AdjustDown(A, i, len);
    }
    
    // 2. 进行 n-1 趟排序
    for(int i = len; i > 1; i--) {
        Swap(A[1], A[i]);      // 堆顶与堆底交换
        AdjustDown(A, 1, i-1); // 把剩余 i-1 个元素重新调整为大根堆
    }
}
```

### 06. 满分兜底：证明与冷门细节
1. **为什么不稳定？（背诵反例）**
   初始序列：`{1, 2, 2_underlined}`
   - 1是根，左右孩子都是2。
   - 代码逻辑中 `if(A[i] < A[i+1])`，当相等时不加加。因此默认选左孩子 `2` 上去。
   - 变成 `{2, 1, 2_underlined}`。经过交换排序后，相对位置必定改变。

2. **如果考卷故意让数组从 `0` 开始存？**
   - 根设为 `i`。
   - 左孩子变为：`2i + 1`
   - 右孩子变为：`2i + 2`
   - 父节点变为：$\lfloor (i-1)/2 \rfloor$
   - 最后一个分支节点：$\lfloor n/2 \rfloor - 1$
   *(默写出公式直接套用，绝不丢分)*
