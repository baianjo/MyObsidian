## 💡 核心考点与功利化总结
**目标定在985，这部分必须做到秒杀：**
1. **最高频考法**：给定初始序列和特定的增量序列（如 $d=4, 2, 1$ 或 $d=3, 1$），要求写出**每一趟排序后的序列状态**。（纯送分，千万别手误）
2. **极易错概念**：希尔排序**仅适用于顺序表**（必须依赖随机访问 `A[i+d]`），**绝对不能**用于链表！
3. **稳定性**：**不稳定**。存在跳跃式移动，相同元素的相对位置极易改变。

---

## 🧠 算法本质：降维打击的插入排序
**前提依据**：直接插入排序在**基本有序**或**数据量小**时效率极高。
**希尔思想**：化全局为局部。先用大增量 $d$ 让序列变得“基本有序”，再逐步缩小 $d$ 逼近全局有序，最后 $d=1$ 时就是一次普通的插入排序（但此时由于基本有序，效率极高）。

### 📈 核心属性（死记硬背）
| 属性 | 结论 | 考研常设陷阱 |
| :--- | :--- | :--- |
| **时间复杂度** | 平均约 $O(n^{1.3})$，最坏 $O(n^2)$ | 考卷若问最坏，答 $O(n^2)$（退化为 $d=1$ 的直接插入）。无法用确切公式证明。 |
| **空间复杂度** | $O(1)$ | 仅需常数级辅助变量（暂存单元）。 |
| **稳定性** | **不稳定** | 跨步长交换，相同元素必定打乱！ |
| **适用结构** | **仅限顺序表** | **选择题高频选项**，由于需跨越式访问 `j-d`，链表无法实现。 |

## 🛠️ 核心操作演示（做题模拟器）
> **考试策略**：不要管代码怎么交替跑，手算做题时，**直接把属于同一个子表的元素抽出来，独立排好序，再塞回原位置**。

**【模拟题】** 对序列 `[49, 38, 65, 97, 76, 13, 27, 49*]`，使用增量序列 $d = 4, 2, 1$ 进行希尔排序。

**第一趟：$d=4$** (距离为4的元素为一组)
*   组1：[**49**, 38, 65, 97, **76**, 13, 27, 49*] $\rightarrow$ 排序：`49, 76`
*   组2：[49, **38**, 65, 97, 76, **13**, 27, 49*] $\rightarrow$ 排序：`13, 38` (换位)
*   组3：[49, 38, **65**, 97, 76, 13, **27**, 49*] $\rightarrow$ 排序：`27, 65` (换位)
*   组4：[49, 38, 65, **97**, 76, 13, 27, **49***] $\rightarrow$ 排序：`49*, 97` (换位)
*   **第一趟结果**：`[49, 13, 27, 49*, 76, 38, 65, 97]`

**第二趟：$d=2$** (基于上一趟结果，距离为2的为一组)
*   组1：[**49**, 13, **27**, 49*, **76**, 38, **65**, 97] $\rightarrow$ 排序 `27, 49, 65, 76`
*   组2：[49, **13**, 27, **49***, 76, **38**, 65, **97**] $\rightarrow$ 排序 `13, 38, 49*, 97`
*   **第二趟结果**：`[27, 13, 49, 38, 65, 49*, 76, 97]`
*(注意：此时可以看到带星号的 49* 已经跑到前面去了，证明了**不稳定**)*

**第三趟：$d=1$** (全局直接插入排序)
*   **最终结果**：`[13, 27, 38, 49, 49*, 65, 76, 97]`

## 💻 必背代码骨架（严蔚敏/王道版）
**考法预警**：大题手撕代码概率极低，但**选择题常考循环变量的运行轨迹**。
注意：机考代码中，为了代码简洁，通常**不会**把一个子表彻底排完再去排下一个子表，而是**交替处理**各个子表（外层循环 `++i` 而不是 `i+=d`）。

```c
void ShellSort(int A[], int n) {
    int d, i, j;
    // 1. 最外层：控制增量 d，每次缩小一半直到等于 1
    for (d = n / 2; d >= 1; d = d / 2) { 
        
        // 2. 中层：交替处理各个子表，从第 d+1 个元素开始
        for (i = d + 1; i <= n; ++i) { 
            
            // 3. 内层：当前元素小于前驱（距离为d），说明需要插入
            if (A[i] < A[i - d]) {
                A[0] = A[i];  // A[0] 仅作为暂存单元（不是哨兵，不能免去越界检查）
                
                // 向前寻找插入位置，步长为 d
                for (j = i - d; j > 0 && A[0] < A[j]; j -= d) {
                    A[j + d] = A[j]; // 记录后移 d 个位置
                }
                A[j + d] = A[0]; // 将暂存值插入正确位置
            }
        }
    }
}
```
**看代码必懂细节（上岸必抠）：**
1. 为什么 `i` 从 `d+1` 开始？ $\rightarrow$ 因为前 $d$ 个元素分别是各个子表的第一个元素，相当于各个子表默认已有序，直接从各子表第二个元素开始考察插入。
2. 为什么是 `++i` 而不是 `i += d`？ $\rightarrow$ 轮流切换处理不同的子表，利用 CPU 缓存局部性，代码写起来也比嵌套三个 `for` 更短。手动算题时不需要用这种思维，直接分组算即可。
