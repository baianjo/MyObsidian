### 1. 核心定义与原理
**本质**：利用**数组**实现的链表。
- **物理结构**：连续内存空间（数组）。
- **逻辑结构**：非连续，通过“游标”链接。
- **游标 (Cursor)**：充当“指针”角色，存储**下一个节点在数组中的下标**。

**内存映射公式**：
若起始地址为 `Addr`，节点大小为 `Size`，则下标为 `k` 的节点物理地址 = `Addr + k * Size`。

**可视化示意**：
```mermaid
graph LR
    A[下标: 0<br>Data: 头结点<br>Next: 2] -->|游标指引| C[下标: 2<br>Data: 元素A<br>Next: 1]
    C --> B[下标: 1<br>Data: 元素B<br>Next: 5]
    B --> D[下标: 5<br>Data: 元素C<br>Next: -1]
    
    style A fill:#f9f,stroke:#333
    style D fill:#bbf,stroke:#333
```
> **特殊标记**：
> *   `Next = -1`：表示链表结束（NULL）。
> *   `Next = -2`（或其他特殊值）：表示节点空闲（脏数据标记，用于分配）。

### 2. 代码定义 (重点考察代码阅读)
考试中可能会遇到两种定义方式，**方式二**较晦涩但课本常用，需看懂。

**方式一：常规结构体数组（易理解）**
```c
struct Node {
    ElemType data;
    int next; // 游标，-1表示结束
};
// 声明一个长度为Max的静态链表
struct Node a[MaxSize]; 
```

**方式二：课本写法（TypeDef 数组别名）**
> **985考点**：理解 `SLinkList` 本身就是数组类型。
```c
#define MaxSize 10
typedef struct {
    ElemType data;
    int next;
} SNode, SLinkList[MaxSize]; 
// 这里的 SLinkList[MaxSize] 等价于定义了一个数组类型

// 使用时：
SLinkList a; // 此时 a 等价于 struct SNode a[MaxSize]
// sizeof(a) = 8 * 10 = 80字节 (假设int=4, data=4)
```

### 3. 关键操作逻辑
| 操作 | 时间复杂度 | 关键步骤 | 与顺序表/单链表对比 |
| :--- | :---: | :--- | :--- |
| **初始化** | $O(n)$ | 1. 头结点 `next` 设为 -1<br>2. 其余空闲节点标记（如 `next = -2`） | 需预处理空闲节点 |
| **查找** | $O(n)$ | 从头结点开始，顺着游标 `next` 找 | **不支持随机存取** (顺序表支持，静态链表不支持) |
| **插入** | $O(1)$* | 1. 找空闲节点 (判 `-2`)<br>2. 修改前驱 `next` 指向新节点<br>3. 新节点 `next` 指向后继 | **无需移动元素** (优于顺序表) |
| **删除** | $O(1)$* | 1. 修改前驱 `next`<br>2. 被删节点标记为空闲 (设 `-2`) | **无需移动元素** (优于顺序表) |

*\*注：插入删除本身是 $O(1)$，但若需先查找位置则总耗时取决于查找。*

### 4. 优缺点与适用场景 (选择题高频)

**优点**：
1.  **增删无需移动元素**：仅需修改游标。
2.  **无需指针支持**：适用于不支持指针的低级语言（如早期Basic、Fortran）。

**缺点**：
1.  **不能随机存取**：必须遍历。
2.  **容量固定**：声明时即固定，**不可扩容**（最大的硬伤）。

**适用场景**：
1.  不支持指针的语言环境。
2.  数据元素数量**固定不变**或变化不大的场景。
