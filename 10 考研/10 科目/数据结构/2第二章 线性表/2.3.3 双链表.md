### 1. 核心定义与结构
**双链表 (Doubly Linked List)**：为了解决单链表查找**前驱结点**麻烦（需要 $O(n)$）的问题，引入前驱指针 `prior`。

**特点**：
- 结点包含三个域：`prior` (前驱指针)、`data` (数据)、`next` (后继指针)。
- **不可随机存取**，按位/按值查找时间复杂度仍为 $O(n)$。
- **对称性**：`p->prior->next == p == p->next->prior`。

```c
typedef struct DNode {
    ElemType data;
    struct DNode *prior, *next; // 前驱、后继指针
} DNode, *DLinkList;
```

### 2. 初始化 (带头结点)
**考点**：头结点的 `prior` 永远指向 `NULL`，头结点的 `next` 指向 `NULL` (若表空)。

```c
bool InitDLinkList(DLinkList &L) {
    L = (DNode *)malloc(sizeof(DNode));
    if (L == NULL) return false;
    L->prior = NULL; // 头结点前驱永远为空
    L->next = NULL;  // 初始为空表
    return true;
}

// 判空条件
bool Empty(DLinkList L) {
    return (L->next == NULL);
}
```

### 3. 核心操作：插入 (后插)
**场景**：在结点 `p` 之后插入结点 `s`。
**严谨逻辑 (不丢分关键)**：必须处理 `p` 是**最后一个结点**的情况，防止空指针解引用错误。

> [!DANGER] 命题陷阱：修改指针顺序
> 必须**先连后断**。如果先执行 `p->next = s`，则原 `p` 之后的结点链就断了（找不到原 `p->next`）。

**标准代码逻辑**：
```c
bool InsertNextDNode(DNode *p, DNode *s) {
    if (p == NULL || s == NULL) return false;
    
    s->next = p->next;      // 1. s 的 next 指向 p 的原后继
    
    // 【核心判空】若 p 是最后一个结点，则无后继，无需修改后继的 prior
    if (p->next != NULL) {  
        p->next->prior = s; // 2. p 的原后继的前驱指向 s
    }
    
    s->prior = p;           // 3. s 的前驱指向 p
    p->next = s;            // 4. p 的后继指向 s
    return true;
}
```
*注：前插操作、按位插入均可转化为“找到前驱结点，执行后插”。*

### 4. 核心操作：删除 (后删)
**场景**：删除结点 `p` 的后继结点 `q`。
**严谨逻辑**：同样需要判断 `q` 是否为最后一个结点。

```c
bool DeleteNextDNode(DNode *p) {
    if (p == NULL) return false;
    DNode *q = p->next;     // q 为要删除的结点
    if (q == NULL) return false; // p 没有后继，无法删除
    
    p->next = q->next;      // 1. p 的 next 指向 q 的后继
    
    // 【核心判空】若 q 是最后一个结点，无需修改其后继的 prior
    if (q->next != NULL) {
        q->next->prior = p; // 2. q 的后继的前驱指向 p
    }
    
    free(q);                // 3. 释放空间
    return true;
}
```

### 5. 遍历与销毁
- **后向遍历**：同单链表。
- **前向遍历**：利用 `prior` 指针。
  - **注意**：跳过头结点。处理数据结点时，循环条件常为 `p->prior != NULL`（若指向头结点则停止）。
- **销毁**：循环执行删除操作，直到表空，最后 `free` 头结点。

### 6. 考研避坑指南 (Summary)
> [!TIP] 高频考点总结
> 1. **代码题手写**：务必加上 `if (p->next != NULL)` 这种判空保护，体现代码健壮性，这是与普通考生的分水岭。
> 2. **选择题**：常考插入/删除的**指针修改顺序**。
>    - 记忆口诀：**先把 `s` 连上两头，最后再改 `p` 的 `next`**。
> 3. **时间复杂度**：
>    - 找前驱：单链表 $O(n)$ vs 双链表 $O(1)$。
>    - 插入/删除：若已知节点位置，均为 $O(1)$。
