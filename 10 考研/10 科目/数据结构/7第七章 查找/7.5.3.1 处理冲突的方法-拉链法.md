## 🚀 核心思想与本质
- **本质**：散列表数组本质上是 **$m$ 个链表的头指针数组**。
- **机制**：将所有散列地址相同（**同义词**）的关键字，通过链表串联起来。
- **计算流程**：$Key \xrightarrow{散列函数 H(key)} 散列地址 \xrightarrow{定位} 对应链表 \xrightarrow{执行增删改查}$。

---

## 🔥 必考操作与满分细节

### 1. 插入操作 (Insert)
> [!tip] 考研默认规则：**头插法**
> 只要题目没特别强调，**一律默认使用链表的头插法**！这意味着后插入的元素在链表最前端。

**操作步骤：**
1. 计算散列地址 $Addr = H(key)$。
2. 实例化新节点，将其通过**头插法**插入到 `Table[Addr]` 指向的链表中。

### 2. 查找操作与查找长度 (Search & ⭐️高频考点)
> [!danger] 绝对不丢分警告：**空指针对比不算查找长度！**
> 考研中，**查找长度 = 关键字的对比次数**。
> 遇到空指针（即链表为空，或遍历到链表尾部的 `NULL`），**对比次数为 0，不计入查找长度**（除非个别自主命题院校特别说明，985统考/常规要求按0算）。

- **查找成功**：遍历链表，找到目标关键字。查找长度 = 该节点在链表中的位序。
- **查找失败**：遍历完整个链表都没找到。查找长度 = **该链表上的节点总数**。

### 3. 删除操作 (Delete)
- **前提**：先查找，找到才能删。
- **操作**：常规的单链表节点删除。
- **注意**：如果删除的是链表中唯一的元素，直接将该位置的头指针置为空（`NULL`）即可。

## 📊 考场实战沙盘推演

**设定条件：** 散列表长度为 13，$H(key) = key \pmod{13}$，使用**头插法**。
**依次插入序列：** 19, 14, 23, 1, 68, 20... (部分示例)
- $H(19)=6$, $H(14)=1$, $H(23)=10$
- $H(1)=1$ （发生冲突！同义词为14。头插法插入到14前面）

**散列表内存快照 (可视化)：**

| 散列地址 (Index) | 头指针 (Head) | 拉链 (同义词链表 - 头插法顺序) |
| :---: | :--- | :--- |
| `0` | `NULL` | |
| `1` | `--->` | `[ 1 ] -> [ 14 ] -> NULL` |
| `...` | `...` | `...` |
| `6` | `--->` | `[ 19 ] -> NULL` |
| `7` | `--->` | `[ 20 ] -> NULL` |
| `8` | `NULL` | |
| `...` | `...` | `...` |
| `10`| `--->` | `[ 23 ] -> NULL` |

**查找长度分析实战：**
1. **查 20**：$H(20)=7$。查地址 7 链表，第 1 个就是。**查找成功，查找长度 = 1**。
2. **查 14**：$H(14)=1$。查地址 1 链表，依次对比 `1`, `14`。**查找成功，查找长度 = 2**。
3. **查 66**：$H(66)=1$。查地址 1 链表，依次对比 `1`, `14` 均不匹配，到达 `NULL`。**查找失败，查找长度 = 2**（只对比了2个实体关键字，NULL不算）。
4. **查 21**：$H(21)=8$。查地址 8，头指针直接为 `NULL`。**查找失败，查找长度 = 0**。

## 🌟 985进阶：拉链法的优化 (用于拉开分差的选择题)

> [!info] 进阶考点：保持链表有序
> 默认的头插法会导致链表内元素**无序**，查找失败时必须遍历整条链表。

**优化方案：** 插入时按**递增/递减有序**排列插入。
**核心优势：提前终止查找，提升“查找失败”时的效率。**
- *场景举例*：链表为递增 `[55] -> [77] -> [99]`。现要查找 `16`。
- *执行逻辑*：对比第一个元素 `55`，发现 $16 < 55$。因为链表递增，后面的元素必然更大，**无需再往后比对**，直接判定查找失败。查找长度缩减。
