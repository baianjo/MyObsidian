> [!danger] 考研功利导向
> 散列（Hash）函数构造在统考及自命题中**极大概率**以选择题或大题第一问出现。
> **核心保分策略**：死磕「除留余数法」中 $p$ 的取值规则，精准记忆其余三种方法的「适用场景匹配」。

## 一、 散列函数设计「四大铁律」
*如果出概念辨析选择题，这四条就是判断对错的唯一标准。*

1. **定义域全覆盖**：必须涵盖所有可能出现的关键字。（反例：对负数开根号报错）
2. **值域不超限**：计算出的地址不能超出散列表合法范围。（反例：表长13，对15取余可能产生下标13、14导致越界）
3. **分布尽可能均匀**：**核心目标**，为了最大程度减少**冲突**。（反例：`key * 13 % 13` 永远等于0，全部冲突）
4. **计算简单快速**：保证查找效率 $O(1)$。（反例：求阶乘计算耗时，拖慢整体效率）

---

## 二、 四大构造方法（考场必杀版）

### 1. 🌟 除留余数法（绝对重点，95%考查概率）
- **公式**：$H(key) = key \pmod p$
- **$p$ 的取值铁律**（**必考**）：$p$ 必须是 **不大于散列表表长 $m$ 的最大质数**。
	- *例：表长 $m=13$，取 $p=13$。表长 $m=15$，取 $p=13$。*
- **为什么必须是质数？**（防大题概念问答）
	- 答：若取合数，关键字与合数的「公因子」会导致散列地址集中在特定位置（如偶数地址），从而加剧冲突。对质数取余能消除公因子影响，使分布更均匀。

**【考场情景复现：合数 vs 质数的分布差异】**
*假设关键字全为偶数：`[2, 4, 6, 10, 12, 14]`，表长为8。*

| 计算方式 | $H(key)$ 计算结果 | 冲突情况 | 结论 |
| :--- | :--- | :--- | :--- |
| **对合数取余** (mod 8) | 2, 4, 6, 2, 4, 6 | ❌ **全部冲突** (仅落在偶数位) | 存在公因子2，分布极度不均 |
| **对质数取余** (mod 7) | 2, 4, 6, 3, 5, 0 | ✅ **零冲突** (散布在0,2,3,4,5,6) | 分布均匀，有效减少冲突 |

### 2. 直接定值法
- **公式**：$H(key) = a \times key + b$ （线性函数）
- **适用场景**：关键字分布 **基本连续**。
- **优缺点**：
	- 优点：绝对不会产生冲突。
	- 缺点：若关键字不连续，会产生大量空位，**严重浪费存储空间**。（例：学号连续可用，但若中间开除几个人，对应位置就永远空着）

### 3. 数字分析法
- **适用场景**：关键字**集合已知**，且某几位**分布均匀**。（例：手机号后4位）
- **做法**：直接抽取分布均匀的数码位作为散列地址。

### 4. 平方取中法
- **适用场景**：关键字的**每一位取值都不均匀/无规律**。
- **做法**：先将关键字平方，再取中间几位作为地址。
- **底层逻辑**（理解即可）：乘法运算是错位相加，中间几位必然受到原关键字**所有数码位**的影响，从而强行“搅拌”出均匀的地址。
	- *视化验证（985平方）：*
	  ```text
	         9 8 5
	       x 9 8 5
	       -------
	       4 9 2 5  <- 受个位5影响
	     7 8 8 0    <- 受十位8影响
	   8 8 6 5      <- 受百位9影响
	   -----------
	   中间位受上述所有行相加的影响
	  ```

## 📝 985上岸速记卡：题眼秒杀匹配

在考研选择题中，看到左侧的**题眼（特征）**，立刻无脑选择右侧的**方法**，绝不丢分：

| 题目给出的关键字特征 | 秒杀对应的构造方法 |
| :--- | :--- |
| 没有任何特殊说明 / 普适场景 | **除留余数法** (找最大质数 $p$) |
| "连续" / "递增" / "线性" | **直接定值法** |
| "已知集合" / "某几位特征明显" (如手机号) | **数字分析法** |
| "分布不均匀" / "无规律" / "位数多" | **平方取中法** |
