## 1. 核心定位与考点
> [!summary] 一句话总结
> **Floyd算法**（弗洛伊德算法）用于求解**各顶点之间**（多源）的最短路径。核心思想是**动态规划**，即“能否通过中转点 $k$ 让 $i$ 到 $j$ 的距离更短”。

*   **适用范围**：
    *   带权图、有向图/无向图。
    *   ✅ **允许负权边**（这是它比Dijkstra强的地方）。
    *   ❌ **不允许负权回路**（会导致路径无穷小，无法求解）。
*   **考研地位**：
    *   **手算**：通常考3-4个顶点的矩阵迭代，考察对“中转点”加入顺序的理解。
    *   **代码**：极简，核心仅5-6行，容易出现在算法填空或简答中。
    *   **复杂度**：时间 $O(\vert V \vert^3)$，空间 $O(\vert V \vert^2)$。

## 2. 算法执行逻辑 (可视化记忆)

我们将求解过程分为 $n$ 个阶段（$n$ 为顶点数）。
令 $A^{(k)}[i][j]$ 表示“中间只允许经过编号 $\le k$ 的顶点”时的最短路径长度。

### 2.1 状态转移方程 (核心)
对于每一个顶点对 $(i, j)$，我们引入中转点 $k$（$k$ 从 $0$ 到 $n-1$ 遍历）：
$$A[i][j] = \min(A[i][j], \quad A[i][k] + A[k][j])$$
*   **直观理解**：是直接从 $i \to j$ 近，还是 $i \to k \to j$ 更近？

### 2.2 矩阵定义
我们需要维护两个矩阵（二维数组）：
1.  **距离矩阵 $A$ (Distance)**：记录当前最短路径长度。
    *   初始化：邻接矩阵（不相连为 $\infty$，自己到自己为0）。
2.  **路径矩阵 $Path$ (Path)**：记录 $i$ 到 $j$ 之间的**最新中转点**。
    *   初始化：全为 -1（表示无中转，直接到达）。

### 2.3 手算技巧 (不丢分关键)
考试手算通常不需要画出所有125次(5^3)运算，只需掌握**更新规则**：

> [!tip] ⚡️ 秒杀技巧
> 在考虑以 $V_k$ 为中转点进行矩阵更新时（第 $k$ 轮）：
> 1.  **第 $k$ 行** 和 **第 $k$ 列** 的元素**永远不会变**（因为 $V_k$ 到 $V_k$ 距离为0，中转自己没意义）。
> 2.  划掉第 $k$ 行和第 $k$ 列，只看剩下的交叉点。
> 3.  **判断逻辑**：$A[i][j]$ 是否大于 $A[i][k] + A[k][j]$？
>     *   即：**当前格子的值 > (同行的"基准列"元素 + 同列的"基准行"元素)**？
>     *   如果是，更新值，并更新 $Path[i][j] = k$。

## 3. 核心代码模板 (必背)

代码极简，三重循环，注意顺序：**k (中转点) 在最外层**。

```c
// 准备工作：A矩阵复制邻接矩阵，Path矩阵全初始化为-1
for (int k = 0; k < n; k++) {          // 1. 最外层：依次尝试所有点作为中转点
    for (int i = 0; i < n; i++) {      // 2. 遍历起点
        for (int j = 0; j < n; j++) {  // 3. 遍历终点
            // 如果经过k中转比原路更近
            if (A[i][j] > A[i][k] + A[k][j]) {
                A[i][j] = A[i][k] + A[k][j]; // 更新距离
                Path[i][j] = k;              // 记录中转点是k (注意这里存的是k)
            }
        }
    }
}
```

## 4. 路径回溯 (难点与坑点)

算出矩阵后，如何还原 $V_i$ 到 $V_j$ 的完整路径？这往往是压轴一问。
*   如果 $Path[i][j] == -1$：说明是直达，路径为 $i \to j$。
*   如果 $Path[i][j] == k$：说明中间经过了 $k$。
    *   **递归逻辑**：路径 = ( $i$ 到 $k$ 的路径 ) + ( $k$ 到 $j$ 的路径 )。
    *   因为 $i$ 到 $k$ 中间可能还有其他点，必须递归查找。

**可视化示例 (V0 -> V4, Path [0][4] =3):**
1.  查 $Path[0][4]=3$ $\Rightarrow$ 路径含 $V_3$: $V_0 \dots V_3 \dots V_4$
2.  递归左半边 $0 \to 3$：查 $Path[0][3]=2$ $\Rightarrow$ 中间有 $V_2$: $V_0 \dots V_2 \dots V_3$
    *   再查 $0 \to 2$：$Path[0][2]=-1$ (直达)
    *   再查 $2 \to 3$：$Path[2][3]=1$ $\Rightarrow$ 中间有 $V_1$: $V_2 \dots V_1 \dots V_3$
3.  最终拼凑：$V_0 \to V_2 \to V_1 \to V_3 \to V_4$

## 5. 三大最短路径算法对比 (考研视角)

| 特性 | **Floyd** | **Dijkstra** | **BFS** |
| :--- | :--- | :--- | :--- |
| **功能** | 多源最短路径 (任意两点) | 单源最短路径 (一点到其他) | 单源最短路径 (仅限无权图) |
| **核心思想** | 动态规划 | 贪心算法 | 广度优先搜索 |
| **适用图** | 带权图，**允许负权边** | 带权图，**不允许负权边** | **无权图** (权值为1) |
| **负权回路** | ❌ 不可解决 | ❌ 不可解决 | ❌ 无意义 |
| **时间复杂度** | $O(\vert V \vert^3)$ | $O(\vert V \vert^2)$ (单次) <br> $O(\vert V \vert^3)$ (n次) | 邻接矩阵: $O(\vert V \vert^2)$ (单次)<br>邻接表: $O(\vert V \vert+\vert E \vert)$ |
| **空间复杂度** | $O(\vert V \vert^2)$ | $O(\vert V \vert)$ | $O(\vert V \vert)$ |
| **代码难度** | ⭐ (最简单) | ⭐⭐⭐ | ⭐⭐ |

> [!failure] 避坑指南
> *   Floyd算法可以解决**负权边**，但不能解决**负权回路**。
> *   若题目问“求各顶点之间最短路径”，首选Floyd；若问“某一点到其他点”，首选Dijkstra。
> *   Dijkstra执行 $n$ 次虽然也是 $O(\vert V \vert^3)$，但无法处理负权边，Floyd更通用。
