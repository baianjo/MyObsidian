> [!NOTE] 核心摘要 (功利化视角)
> 1. **考研重点**：图的基本操作是**BFS (广度优先遍历)** 和 **DFS (深度优先遍历)** 的基石。
>     
> 2. **不丢分关键**：熟练掌握 **邻接矩阵 (Array)** 与 **邻接表 (Linked List)** 在不同操作下的**时间复杂度差异**，特别是**有向图的入边**处理。
>     
> 3. **常考接口**：`FirstNeighbor` (找第一个邻接点) 和 `NextNeighbor` (找下一个邻接点) 是手写代码题的核心依赖。
>     

### 一、 核心操作复杂度对比 (背诵级)

这是本节最“功利”的部分，直接决定选择题能否秒杀以及大题能否写出最优解。

**符号约定**：$V$ = 顶点数 (Vertex)，$E$ = 边数 (Edge)。

#### 1. 查找与判断类 (Read-Only)

|**操作**|**描述**|**邻接矩阵 (Dense优化)**|**邻接表 (Sparse优化)**|**备注 (避坑点)**|
|---|---|---|---|---|
|**Adjacent(x, y)**|判断边 $<x, y>$ 是否存在|**$O(1)$**|$O(1) \sim O(\vert V \vert)$|矩阵直接索引；邻接表需遍历 x 的边链表 (最坏情况 x 连满所有边)。|
|**Neighbors(x)**|列出 x 的所有邻接边|**$O(\vert V \vert)$**|**出边：$O(1) \sim O(\vert V \vert)$**<br><br>  <br><br>**入边(有向)：$O(\vert E \vert)$**|矩阵必须扫描整行；邻接表找**有向图入边**极慢，除非用逆邻接表。|
|**FirstNeighbor(x)**|找 x 的第一个邻接点|**$O(1) \sim O(\vert V \vert)$**|**$O(1)$**|矩阵最坏需扫描整行才找到第一个；邻接表直接访问链表头。|
|**NextNeighbor(x, y)**|找 x 邻接点 y 后的下一个|**$O(1) \sim O(\vert V \vert)$**|**$O(1)$**|遍历算法的核心。矩阵需继续扫描行；邻接表只需 `p->next`。|

#### 2. 修改类 (Write)

| **操作**               | **描述**       | **邻接矩阵**               | **邻接表**                      | **备注 (避坑点)**                                        |
| -------------------- | ------------ | ---------------------- | ---------------------------- | --------------------------------------------------- |
| **InsertVertex(x)**  | 插入新顶点 x      | **$O(1)$**             | **$O(1)$**                   | 矩阵：在数组末尾追加（假设未满）；邻接表：在顶点数组末尾追加。                     |
| **DeleteVertex(x)**  | 删除顶点 x 及相关边  | **$O(\vert V \vert)$** | **$O(\vert E \vert)$**       | 矩阵：**清空**对应的行和列（置0或标记），通常不移动数据。邻接表：需遍历整个图删除指向 x 的边。 |
| **AddEdge(x, y)**    | 新增边 $<x, y>$ | **$O(1)$**             | **$O(1)$**                   | 矩阵：赋值为1；邻接表：**头插法**插入链表 (最快)。                       |
| **RemoveEdge(x, y)** | 删除边 $<x, y>$ | **$O(1)$**             | $O(1) \sim O(\vert V \vert)$ | 矩阵：赋值为0；邻接表：需遍历链表找到结点并删除。                           |

### 二、 深度解析与避坑指南 (985上岸必看)

#### 1. 为什么 `DeleteVertex` 这么麻烦？

- **邻接矩阵**：
    
    - **低效做法**：删除数组中间元素，后面元素前移。代价巨大，除了 $O(\vert V^2 \vert)$ 的数据移动，还要维护顶点编号映射，**考研不推荐**。
        
    - **高效做法 (推荐)**：将该顶点标记为 `Empty` (或布尔变量 `isValid = false`)，并将其对应的行列清零。时间复杂度 $O(\vert V \vert)$。
        
- **邻接表**：
    
    - 删除顶点 `x` 本身很快，但在 **无向图** 或 **有向图** 中，必须把**其他顶点指向 `x` 的边**也删掉。
        
    - 这意味着需要扫描其他所有顶点的边链表，最坏情况要遍历所有的边，故复杂度为 $O(\vert E \vert)$。
        

#### 2. 有向图的“入边”噩梦

- 在 **邻接表** 中，找 `x` 指向了谁（出边）很容易。
    
- 但要找 **谁指向了 `x` (入边)**，必须遍历整个邻接表的所有边节点。
    
- **考研对策**：如果题目频繁涉及查找入边，**不要选普通邻接表**，考虑使用 **邻接矩阵** 或 **逆邻接表 (Inverse Adjacency List)**。
    

#### 3. 遍历算法的底层依赖

图的遍历 (BFS/DFS) 实际上就是不断调用 `FirstNeighbor` 和 `NextNeighbor` 的过程。

- **邻接矩阵**：由于这两个操作可能需要扫描无效的 `0` 元素，导致遍历复杂度为 $O(\vert V^2 \vert)$。
    
- **邻接表**：由于直接跳过不存在的边，遍历复杂度优化为 $O(\vert V \vert + \vert E \vert)$。
    
- **结论**：**稀疏图 (Sparse Graph)** 选邻接表，**稠密图 (Dense Graph)** 选邻接矩阵。

### 三、 核心代码逻辑 (伪代码)

考研不需要你默写所有基本操作，但必须会调用。以下是你在手写算法题中经常要用到的逻辑模式。

#### 1. 找第一个邻接点 `FirstNeighbor(G, x)`

- **邻接矩阵**：
    
    C++
    
    ```
    // 扫描第 x 行，找到第一个非0 (或非无穷) 元素
    for (int j = 0; j < G.vexnum; j++) {
        if (G.Edge[x][j] != 0) return j;
    }
    return -1;
    ```
    
- **邻接表**：
    
    C++
    
    ```
    if (G.vertices[x].firstarc != NULL)
        return G.vertices[x].firstarc->adjvex;
    return -1;
    ```
    

#### 2. 找下一个邻接点 `NextNeighbor(G, x, y)`

- **邻接矩阵**：
    
    C++
    
    ```
    // 从 y+1 开始继续扫描第 x 行
    for (int j = y + 1; j < G.vexnum; j++) {
        if (G.Edge[x][j] != 0) return j;
    }
    return -1;
    ```
    
- **邻接表**：
    
    C++
    
    ```
    // 在链表中找到 y 节点，返回它的 next
    ArcNode *p = G.vertices[x].firstarc;
    while (p != NULL && p->adjvex != y) p = p->next;
    if (p != NULL && p->next != NULL)
        return p->next->adjvex;
    return -1;
    ```

### 四、 总结与下一步

- **基本操作本质**：邻接矩阵是 **“空间换时间”** (查找快，存稀疏图浪费)，邻接表是 **“时间换空间”** (存稀疏图省空间，找入边慢)。
    
- **下一步建议**：
    
    这些基本操作是工具，目的是为了构建更复杂的算法。建议利用这两个操作的特性，开始复习 **图的遍历 (BFS/DFS)**，并尝试分析为什么 BFS 需要辅助队列，而 DFS 需要递归栈。
