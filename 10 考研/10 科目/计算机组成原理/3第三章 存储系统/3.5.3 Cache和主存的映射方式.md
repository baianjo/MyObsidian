# 🚀 【考研/408绝杀】Cache与主存的三种映射方式

> **💡 导语（致985人）：**
> 这一块是计组**必考、常考大题**的送分/丢分重灾区。
> 别管教材说得多玄乎，考研对这块的考核极其功利：**就是考你“怎么把一个给定的主存地址，切成几段，去Cache里找对应的数据”**。掌握“切面包”大法，这部分你一分都不会丢。

---

## ⚠️ 绝对不能丢分的大前提（闭眼都要记住的公理）
1. **块大小绝对相等：** `Cache行(块)大小 = 主存块大小`。数据在主存和Cache之间搬运，永远以**“块”**为基本单位。
2. **一块空间不只有数据：** 站在计算机硬件（0和1）的角度，Cache初始全是0，所以必须增加 **1位的有效位（Valid Bit）**。
   - 考点坑位：如果题目问“Cache的**总容量**”，别只算存数据的容量！
   - `Cache总行容量 = 1位有效位 + 标记位(Tag)长度 + 数据块大小` （有时还会加上脏位/替换算法控制位，看题目条件，但**有效位必有**）。

---

## 🛠️ 实战演示：一套数据贯穿三种映射
> **🎯 考试通用场景（源自视频示例）：**
> - **主存容量：** 256MB ($2^{28}$ 字节) $\Rightarrow$ **主存地址总长度 = 28位**
> - **按字节编址**
> - **Cache/主存块大小：** 64B ($2^6$ 字节) $\Rightarrow$ **块内地址 = 6位**
> - **主存块数：** $2^{28} \div 2^6 = 2^{22}$ 块 $\Rightarrow$ **主存块号 = 22位**
> - **Cache总行(块)数：** 8行 ($2^3$ 行)

我们来看看，对于同一个主存地址，CPU采用不同映射方式时，是怎么“切”这28位地址的。

### 1. 全相联映射 (Fully Associative) —— “海王模式：哪有空位去哪”
* **说人话：** 主存的任何一个块，可以随便塞进Cache的任何一行。
* **找数据方式：** 拿着主存块号，和Cache里所有行的标记（Tag）进行**逐一对比**（现实中是并发比较）。
* **地址切分：** 纯两段式。
  `[ 标记 Tag (22位) ]  [ 块内地址 (6位) ]`
  *(注：这里的Tag就是完整的主存块号)*
* **优缺点及考点：**
  - **优：** 空间利用率最高，命中率最高（只要Cache没满就不会发生冲突）。
  - **缺：** 查找最慢（得对比所有的Tag），硬件成本极高。
  - **考点：** 只有全相联的Tag等于完整主存块号。

---

### 2. 直接映射 (Direct Mapping) —— “社恐模式：只坐专属座位”
* **说人话：** 每个主存块只能存放在Cache中**唯一固定**的行。
* **怎么定座位？** `主存块号 % Cache总行数`
  - 硬件不会做除法，因为Cache行数一定是 $2^n$（本题是 $8 = 2^3$）。
  - **取余 = 截取主存块号的最后 $n$ 位！** （本题中就是主存块号的最后3位，作为Cache行号）。
  - 既然最后3位已经用来找行号了，这3位就不用存到标记（Tag）里了！
* **地址切分：** 三段式。
  `[ 标记 Tag (19位) ]  [ Cache行号 Index (3位) ]  [ 块内地址 (6位) ]`
  *(算法：28位总长 - 6位块内 - 3位行号 = 19位Tag)*
* **优缺点及考点：**
  - **优：** 查找最快（根据中间3位直接定位到唯一一行，比对1次Tag就行）。
  - **缺：** 极其容易发生冲突（即使Cache其他行空着，只要有两人抢同一个专属座位，也得把前一个人踢走），命中率最低。

---

### 3. 组相联映射 (Set-Associative) —— “中庸之道：先定包厢，随便坐位” (最常考⭐)
* **说人话：** 把Cache分组。主存块通过取余决定进哪个**组**，进了组之后，组内的座位**随便挑**。
* **考点术语解析：`n路组相联`** = **每组有n个Cache行**。
  - 本题Cache共8行。如果题目说是**2路组相联**（每组2行）。
  - **分组总数** = 8行 $\div$ 2行/组 = **4组** ($2^2$ 组)。
* **怎么定包厢？** `主存块号 % 组数`
  - 同理，截取主存块号的最后 $k$ 位（分组数 $= 2^k$，本题是4组，截取最后2位作为组号）。
* **地址切分：** 三段式。
  `[ 标记 Tag (20位) ]  [ 组号 Set Index (2位) ]  [ 块内地址 (6位) ]`
  *(算法：28位总长 - 6位块内 - 2位组号 = 20位Tag)*
* **优缺点：**
  - 结合了前两者的优点，命中率较高，查找速度适中。
  - **极端情况：** 1路组相联 = 直接映射；全容量路数（如8行就是8路）组相联 = 全相联映射。

## 🧠 985高阶必会：CPU访存(Hit/Miss)真实流程可视化
不管哪种映射，如果你是一颗CPU，拿到一个28位地址，你的操作必须遵循以下铁律流水线（考研大题必写步骤）：

1. **切地址：** 根据映射方式，把28位二进制切成对应的 `[Tag] [Index] [块内地址]`。
2. **找位置：** 看 `Index` 段，直接跳到Cache对应的行/组。
3. **验身份（2个条件缺一不可，绝杀点）：**
   - 提取该Cache行的 `有效位(Valid Bit)`，看是否为 `1`。
   - 提取该Cache行的 `Tag`，与地址里的 `Tag` 段进行严格的**等值对比**。
4. **出结果：**
   - **命中 (Hit)：** 有效位为1 **且** Tag匹配！直接根据 `块内地址` 从这行提取目标字节。
   - **未命中 (Miss)：** 有效位为0 **或** Tag不匹配。CPU老老实实去主存读对应的整个主存块，搬运到Cache中（如果位置被占了，还需要根据LRU等算法进行替换）。

## 📊 一图流总结大抄表（建议默写）
*(假设：主存地址位数 $M$，Cache总行数 $2^c$，每组行数 $2^n$，块大小 $2^b$)*

| 映射方式 | 块内地址位数 | 索引段 (Index) 是什么？ | 索引段位数 | 标记位 (Tag) 位数 (不含有效位) |
| :--- | :--- | :--- | :--- | :--- |
| **全相联** | $b$ | 无 (直接全场乱找) | 0 | $M - b$ |
| **直接映射** | $b$ | **Cache行号** | $c$ | $M - c - b$ |
| **组相联** | $b$ | **组号** | $c - n$ | $M - (c - n) - b$ |

> **“无论怎么变，块内地址永远在最后并且长度雷打不动；剩下部分怎么分给Tag和Index，全看Cache切成了多少个坑位！”**
