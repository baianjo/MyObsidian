# 🚀 SRAM 与 DRAM 核心突击指南（满分无死角版）

> [!abstract] 提纲挈领
> 这一节是计组**必考、常考**的选择题/大题考点。你不需要会造芯片，你只需要记住它俩的**物理天性**，就能秒杀所有特性对比题。
> - **SRAM（静态RAM）** = **贵族**：用钱（晶体管）砸出来的速度，快、贵、发热、不需要操心（不用刷新），用来做 **Cache**。
> - **DRAM（动态RAM）** = **打工人**：便宜、量大、干活慢（电容充放电），记性还差（需要一直刷新），用来做 **主存（内存条）**。

---

## 🆚 核心对比：一表背诵（选择题必考）

> [!important] 必背表格
> 考试遇到对比题，直接在脑子里调取这张表，**绝对不会丢分**。

| 对比维度 | **SRAM（静态）** | **DRAM（动态）** | 🧠 怎么记（人话速记） |
| :--- | :--- | :--- | :--- |
| **存储元** | **双稳态触发器** (6个MOS管) | **栅极电容** (1个电容+1个MOS) | SRAM零件多(贵)，DRAM零件少(便宜) |
| **主要用途** | **Cache（高速缓存）** | **主存（内存）** | 贵的离CPU近，便宜的做大容量 |
| **读出方式** | **非破坏性读出** | **破坏性读出** | 读DRAM相当于给电容放电，读完数据就没了！ |
| **读后重写** | 不需要 | **需要（读后再生）** | 既然DRAM读完就没，必须重新写回去（导致慢） |
| **运行速度** | 快 | 慢 | DRAM要重写，当然慢 |
| **集成度** | 低 | **高** | DRAM零件小，同样面积能塞更多，容量大 |
| **功耗** | 大 | 小 | SRAM有6个管子在耗电，发热大 |
| **是否刷新** | 不需要 | **需要** | **DRAM的电容会漏电**，必须定期充电（刷新） |
| **地址线送法** | 行、列地址**同时送** | 行、列地址**分两次送（地址复用）** | **极易错考点**！DRAM地址引脚数量会**减半**！ |

> [!warning] 致命易混淆点（985常设陷阱）
> 1. **破坏性读出 vs 易失性**：
>    - **易失性**：断电数据就丢。**SRAM 和 DRAM 都是易失性存储器！**（断电都变砖）。
>    - **破坏性读出**：没断电，仅仅是读一次数据，数据就被破坏了。**只有DRAM是破坏性读出！**
> 2. **现代内存**：平时买电脑说的 DDR3, DDR4 全称叫 **SDRAM**（同步动态随机存储器），本质还是 DRAM。

## 🔄 DRAM的刷新机制（重难点）

**为什么只有DRAM需要刷新？**
因为DRAM用**电容**存数据，电容即使不断电也会**漏电**。通常电容最多只能撑 **$2ms$**，所以必须在 $2ms$ 内给它重新充满电（即刷新）。

### 1. 刷新的 3 个铁律（一分不能丢）
- **刷新单位**：以 **行** 为单位。每次刷新一整行（利用内部专门的刷新电路）。
- **刷新耗时**：刷新一次的耗时 = 1个读写周期（存取周期）。本质是一次“只读不输出”的操作。
- **谁来控制**：刷新由**存储器独立完成**，**不需要CPU介入**！（考题常骗你说CPU控制刷新，打死别信）。

### 2. 三种刷新策略对比（可视化）
假设：存储器有 128行，存取周期为 $0.5\mu s$，要求 $2ms$ 内全部刷新完。

#### ① 集中刷新（一刀切，不推荐）
- **做法**：前一段时间正常读写，最后抽出一段时间，把128行一口气全刷了。
- **致命缺点**：有一段很长的**死时间（死区）**。在这段时间内，CPU绝对无法访问内存。

#### ② 分散刷新（强迫症，太费时）
- **做法**：每次读/写完一次，立刻强制刷新一行。也就是 `工作周期 = 1次正常读写 + 1次刷新`。
- **缺点**：导致存取周期直接翻倍（变慢），而且刷新过度频繁，没必要。**无死区**。

#### ③ 异步刷新（最科学、最常考）
- **做法**：把 128 次刷新操作，**均匀平摊**到 $2ms$ 内。
- **公式**：`相邻两次刷新的时间间隔 = 最大刷新周期(2ms) / 行数(128)` = $15.6\mu s$。
- **效果**：每隔 $15.6\mu s$ 才会有一次 $0.5\mu s$ 的死时间。
- **工程妙招**：聪明的系统会把这 $0.5\mu s$ 安排在 CPU 内部译码阶段（此时CPU本来就不需要访问内存），完美隐藏死时间！

## 📍 地址线复用技术（计算题/引脚数杀手）

**背景**：如果存储器有 $2^n$ 个存储单元，如果排成一排，译码器极其庞大。所以工程上排成**二维矩阵**，把 $n$ 位地址拆成：**行地址 ($n/2$ 位) + 列地址 ($n/2$ 位)**。

> [!tip] DRAM 的地址复用（必考计算）
> 因为 DRAM 容量极大，如果全接地址线，引脚太多装不下。
> **DRAM 做法**：**同一组地址引脚，用两次！** 
> 第一次传行地址，第二次传列地址。

**📌 考试怎么拿分？记住这个推导公式：**
假设题目说某 DRAM 芯片容量为 $1024 \times 1024$（即 $1M$ 容，也就是 $2^{20}$ 地址）：
1. **理论地址线总数**：$n = 20$ 根。
2. **实际物理地址引脚数**：由于分两次送，引脚数直接**减半**！即 $20 / 2 = 10$ 根引脚！

**📝 经典例题演示**：
问：某 DRAM 芯片容量为 $4M \times 8$ 位，求该芯片的地址引脚和数据引脚数量？
- **解答**：
  - 数据引脚：看后面，8位就是 **8根** 数据线。
  - 容量 $4M = 2^{22}$，理论需要22根地址线。
  - 因为是 **DRAM（默认地址复用）**，地址引脚减半：$22 / 2 =$ **11根**！
  - 总引脚数 = $11 + 8 = 19$根（可能还要加电源、地线、片选等，看题目具体问法）。

*注：如果是 SRAM $4M \times 8$位，地址不复用，地址引脚就是老老实实的 22 根！*
