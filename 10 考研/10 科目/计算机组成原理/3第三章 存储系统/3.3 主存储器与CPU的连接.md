# 主存储器与CPU的连接（扩展与译码）

> 💡 **大白话引言**：
> 现在的CPU胃口很大（比如一次要吞吐64位数据，能识别几百GB的地址），但我们去进货买到的**单块存储芯片**往往很小（比如每次只能吞吐8位，容量只有8KB）。
> 怎么让这些“小芯片”满足“大CPU”？**核心思想就是“拼积木”**。缺数据位宽就拼位宽（位扩展），缺容量就拼容量（字扩展），全缺就全拼（字位同时扩展）。

---

## 一、 考前必知：单块芯片的“黑话”与引脚
在拼积木之前，先搞懂一块芯片有哪些接口要和CPU连。考试中常以英文缩写出现，**必须秒懂**：

1. **地址线 (Address, $A$)**：$A_0 \sim A_n$。如果有13根线（$A_0 \sim A_{12}$），说明芯片有 $2^{13} = 8K$ 个存储单元。**地位连低位，高位连高位**。
2. **数据线 (Data, $D$)**：$D_0 \sim D_m$。如果有8根线，说明芯片字长是8位（1 Byte）。
3. **片选线 (Chip Select, $CS$ 或 $CE$)**：相当于芯片的“工作开关”。
   - **没横线 (CS)**：高电平（1）有效。
   - **有横线 ($\overline{CS}$)**：低电平（0）有效。（电路图上常在引脚处画个**小圆圈**代表低电平有效，考试必考看图识电平）。
4. **读写控制线 ($WE$ 或 $WR$)**：
   - 如果一根线控制：$\overline{WE}$（低电平0为写 Write，高电平1为读 Read）。
   - 如果两根线分开控制：$\overline{WE}$管写，$\overline{OE}$管读（均低电平有效）。

## 二、 核心考点：三大扩展法（绝对不丢一分）

### 1. 位扩展法（只加宽，不增高）
- **适用场景**：芯片容量够了，但字长比CPU的数据总线窄。（例：用 8K×1位的芯片，去适配 64位数据总线的CPU）。
- **人话图解**：假设8片 8K×1位 芯片。CPU给出一个地址，**8块芯片同时开工**，每块贡献1个比特，刚好凑齐8个比特给CPU。
- **接线秘籍**：
  - **地址线**：全部并联（CPU的 $A_0 \sim A_{12}$ 同时接给所有芯片）。
  - **片选线 / 读写线**：全部并联（要干活一起干活）。
  - **数据线**：**分别接入**（第1块接 $D_0$，第2块接 $D_1$ ... 第8块接 $D_7$）。

### 2. 字扩展法（只增高，不加宽）
- **适用场景**：芯片的字长已经和CPU一致了，但是容量太小，CPU的高位地址线闲置了。（例：用 8K×8位的芯片，拼出 16K×8位的内存）。
- **人话图解**：多块芯片轮流干活。CPU给个低位地址找具体位置，用**高位地址**来决定到底是哪块芯片（哪层楼）出来接客。
- **两种选片方案（必考对比）**：
  1. **线选法（直接用高位地址线连CS）**
     - *做法*：$A_{13}$ 连第一块， $A_{14}$ 连第二块。
     - *致命缺点*：地址空间**不连续**！如果 $A_{13}$ 和 $A_{14}$ 同时为1（或同时为0），就会发生冲突（两块芯片同时往总线发数据）或都不工作。**实际不用，仅作考题干扰项。**
  2. **译码器片选法（985必考！完美方案）**
     - *做法*：把CPU多出来的 $n$ 根高位地址线，接进一个译码器，翻译出 $2^n$ 个状态，分别连接 $2^n$ 块芯片的CS端。
     - *优点*：地址空间绝对**连续**。输入二进制 `00` 选第0片，`01` 选第1片，绝不冲突。

### 3. 字位同时扩展法（又加宽，又增高）
- **适用场景**：字长和容量都不够。
- **做题/画图步骤（严格按此逻辑）**：
  1. **先做位扩展，化零为整**：比如用两块 16K×4位 芯片，先并联成一组，变成一个“**逻辑上的 16K×8位 大芯片（Group）**”。
  2. **再做字扩展，组间叠高**：把刚才拼好的这些 Group，利用高位地址线+译码器，分配给不同的片选信号。
  - *注意*：同一个 Group 内的芯片，片选信号必须接在**同一个**译码器输出口上！（同生共死）

## 三、 进阶硬核考点：译码器与时序（拉开分差的985专属地带）

对于跨考或没学过数电的同学，这里最容易丢分。请死记以下原理：

### 1. 常见的译码器 (如 74LS138)
- **输入输出对应**：
  - **1-2译码器**：1输入，2输出 ($2^1$)。
  - **2-4译码器**：2输入，4输出 ($2^2$)。
  - **3-8译码器 (重点 74LS138)**：3输入 ($A,B,C$)，8输出 ($Y_0 \sim Y_7$)。
- **电平极性匹配（踩坑点）**：
  - 如果译码器输出端画了小圆圈（低电平有效输出 0），那么它必须配合**低电平有效 ($\overline{CS}$)** 的存储芯片！
  - 考题若给出高电平有效的芯片，必须确保译码器输出也是高电平有效（或中间加非门）。

### 2. 译码器的“使能端 (Enable, $EN / G$)” 与 时序问题
真实的译码器不仅有地址输入，还有**使能端**（相当于译码器的总开关）。比如 74LS138 通常有3个使能端（如 $G_1$ 高有效，$G_{2A}$ 低有效，$G_{2B}$ 低有效，必须全部满足条件它才干活）。

- **为什么需要使能端？（究极考点：消除地址毛刺）**
  1. CPU发出地址电信号时，电流会有短暂的**不稳定期（毛刺）**。如果立刻让译码器工作，可能会选中错误的芯片，读写错误的数据。
  2. CPU有一个绝招：发出**主存请求信号 $\overline{MREQ}$** (Memory Request)。
  3. **工作流程（时序）**：
     - **第一步**：CPU先送出稳定的 $A_0 \sim A_n$ 地址信号。
     - **第二步**：CPU原地等一会儿（让电信号飞一会儿，稳定下来）。
     - **第三步**：CPU发出 $\overline{MREQ}$ 信号。
     - **第四步**：把 $\overline{MREQ}$ 连到**译码器的使能端**。此时译码器才通电开始翻译，发出片选信号 $CS$。
     - **第五步**：芯片被选中，开始输出/写入数据。数据传送完毕后，先撤销 $\overline{MREQ}$（关芯片），最后再撤销地址信号。

> ⚠️ **大题/选择题防坑雷达**：
> 如果考到时序图，记住顺序永远是：**地址线波形先变（送地址） -> 等待极短时间 -> 片选线波形变（发出请求选中） -> 数据线波形变（出数据）**。

## 四、 快速做题秒杀总结（无脑计算流）

遇到“需要多少块芯片”、“地址范围是什么”的题，直接套格式：

*表格说明：使用 `/vert` 防止冲突*
1. **求芯片总数**：
   总芯片数 = (目标总字数 / 单片字数) $\times$ (目标位宽 / 单片位宽)
   *例：用 8K×1位 组成 32K×8位主存。芯片数 = (32K/8K) $\times$ (8/1) = 4 $\times$ 8 = 32片。*

2. **划分地址引脚（超高频）**：
   CPU总地址线 = $N$ 根（比如16根，寻址 $64K$）。
   单块芯片容量 = $2^k$（比如 $8K$，即 $2^{13}$，所以芯片内部需要 $13$ 根地址线）。
   - **低位接芯片**：$A_0 \sim A_{k-1}$ （例：$A_0 \sim A_{12}$）连接所有芯片的地址引脚。
   - **高位接译码器**：剩余的高位线 $A_k \sim A_{N-1}$ （例：$A_{13} \sim A_{15}$）接译码器，用于产生片选信号。

3. **求某片芯片的地址范围**：
   把接在译码器上的高位地址线固定为特定二进制组合，低位地址线全部填 `0`（起点）到全部填 `1`（终点），转换成十六进制即为地址范围。
   *例：选第0组，高位 $A_{15}A_{14}A_{13} = 000$，低位13位全0到全1。*
   起点：`000` `0000 0000 0000` = `0000H`
   终点：`000` `1111 1111 1111` = `1FFFH`
