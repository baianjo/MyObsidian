> [!abstract] 考点本质 (直击130分核心)
> DMA (直接内存存取) 方式实现了 **CPU 与外设的“成块数据完全并行”**。
> 408 核心考点：**DMA 控制器的核心寄存器分工（尤其是字计数器 WC 溢出中断）、三种总线权争夺方式（重点掌握“周期挪用”）、以及 DMA 与程序中断方式的十维全方位对比（高频单选/大题）**。

### 一、 DMA 控制器的内部物理结构

DMA 控制器（DMAC）是一个挂在系统总线上的硬核硬件，它内部包含以下关键寄存器：

```
                        [ 系统总线 ]
                         |  |  |
        +----------------+  |  +---------------+
        |                   |                  |
        v                   v                  v
  [ 传送长度计数器 WC ]  [ 主存地址寄存器 AR ]  [ 数据缓冲 DBR ]
    (存传送字数补码)      (存当前主存物理地址)     (暂存传输数据)
```

*   **主存地址寄存器 (AR)**：存放要读写的主存单元物理地址。每传输完一个字，AR 自动自增。
*   **外设地址寄存器 (FAR)**：存放外设的端口号或内部地址。
*   **传送长度计数器 (WC, Word Count)（🚨 🚨 🚨 408 核心考点）**：
    用来记录还要传输多少个字。
    *   **物理实现**：**WC 默认存入传送字数的“双字长补码”**。
    *   **工作机制**：每成功传输一个字，WC 寄存器自动执行 **$+1$**。当 WC 累加到发生**溢出（即值变成全 0）**时，说明所有数据传输完毕。此时，控制逻辑会**触发一个溢出脉冲，向 CPU 发出中断请求**，通知 CPU 传输结束。
*   **数据缓冲寄存器 (BR)**：暂存正在传输的单字数据。

### 二、 三种 DMA 传送方式（总线使用权争夺）

由于 DMA 控制器和 CPU 共享同一根系统总线，当它们同时想访存时，如何协调总线使用权？

#### 1. 停止 CPU 访存（CPU 挂起法）
*   **机制**：一旦 DMA 控制器申请总线，CPU 立即交出总线控制权，停止执行程序并挂起。直到一整块数据全部传完，DMA 才归还总线。
*   **优缺点**：控制极简，但 CPU 闲置时间极长，完全没有发挥出并行的优势。

#### 2. 周期挪用（周期窃取 / Cycle Stealing——高频重点！）
*   **机制**：**化整为零，每次只借一个周期**。外设每准备好一个字，DMA 就申请占用总线**一个存取周期（访存周期）**，传输这一个字，传完立即归还。
*   **冲突分析（CPU 与 DMA 狭路相逢）**：
    *   **场景 A**：CPU 此时正好不需要访存。DMA 默默借用，**CPU 零开销，双赢**。
    *   **场景 B**：CPU 此时也想访存。**DMA 享有绝对优先权**！CPU 必须等一个存取周期，让 DMA 先传。
        *(注：因为外设输入的数据流如果不在数据缓冲溢出前读走，新来的数就会把旧数覆盖掉，造成数据永久丢失；而 CPU 的指令可以无限期推迟执行。)*

#### 3. DMA 与 CPU 交替访存（分时分块访存）
*   **机制**：把一个系统时钟周期平分为两半，前半周期给 CPU 专用，后半周期给 DMA 专用。
*   **特点**：总线控制权交接由硬件时序固化，不需要申请和释放过程。CPU 和 DMA 各走各的路，互不干扰，但对硬件总线物理频率要求极高。

### 三、 DMA 方式 vs 程序中断方式（408 终极对比表👑）

这是 408 无论选择题还是大题中，几乎每隔一年就要拉出来“拉踩”一次的终极黄金考点：

| 对比维度 | 程序中断控制方式 | DMA 控制方式 |
| :--- | :--- | :--- |
| **数据传送单位** | **单字节** 或 **单字** | **数据块** (通常包含成百上千个字) |
| **响应时机** | **一条指令执行结束时** (指令周期结束时)<br>此时检测中断请求。 | **一个存取周期结束时** (机器周期/总线事务结束时)<br>不影响当前指令执行。 |
| **传送路径** | **外设 ➜ CPU 寄存器 ➜ 主存**<br>(必须经过 CPU 寄存器中转) | **外设 ➜ DMA ➜ 主存**<br>(直接连通，完全绕过 CPU) |
| **硬件与软件开销** | 硬件开销小，主要靠**软件（中断服务程序）**搬运。 | 硬件开销极大，必须配有专用的 **DMA 控制器**。 |
| **优先级** | 优先级**低**。 | 优先级**极高** (高于所有中断源，防止丢数据)。 |
| **中断的作用** | **实际搬运数据**。<br>CPU 被叫过去干“搬运工”的脏活累活。 | **仅仅用于传输结束后的收尾校验**。<br>CPU 只来主持最后的剪彩仪式。 |
| **典型应用场景** | 慢速、突发性事件 (如键盘、鼠标、异常处理)。 | 高速、成块的数据传输 (如机械硬盘、固态硬盘、网卡)。 |

---

### 🚨 避坑警告：DMA 过程中的两次中断区分
408 大题非常喜欢考细节：**“DMA 传送过程中，CPU 到底响应了几次中断？”**
*   **答**：**只有一次**！就是在数据块全部传输完毕，WC 发生溢出时，由 DMA 控制器向 CPU 发送的“传送结束中断”（用于后处理）。
*   在 DMA 准备阶段（预处理）和实际传输中，**完全没有中断发生**！
