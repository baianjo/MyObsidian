### 一、 I/O 接口的物理结构与端口

I/O 接口（如网卡、声卡控制芯片）内部最核心的是三个寄存器（统称为 **I/O 端口**）：

```
                       [ 系统总线 ]
                        |  |  |
        +---------------+  |  +---------------+
        |                  |                  |
        v                  v                  v
  [ 控制寄存器 CR ]  [ 状态寄存器 STR ]  [ 数据缓冲 DBR ]
    (CPU只写命令)      (CPU只读状态)       (双向暂存数据)
```

> [!tip] 985 高分必杀技：端口地址的“重叠艺术”
> 状态寄存器（STR）是**只读**的，控制寄存器（CR）是**只写**的。
> 在物理设计上，为了节省珍贵的地址线，**可以让 STR 和 CR 共享同一个物理端口地址**。
> CPU 发出 `读` 该地址时，读出的是外设的状态；发出 `写` 该地址时，是将命令写入控制寄存器。

### 二、 I/O 端口两大编址方式（选择题超级重点！）

CPU 如何在指令中区分“这是一个内存地址”还是“这是一个 I/O 端口地址”？有以下两种核心编址逻辑：

| 编址方式 | 英文与别名 | 工作原理与物理特征 | 优缺点与指令特征 |
| :--- | :--- | :--- | :--- |
| **统一编址** | Memory-mapped I/O<br>(内存映射 I/O) | 把 I/O 端口和主存单元**混在一起编址**。<br>比如 $0 \sim 3GB$ 给内存，$3GB \sim 4GB$ 划归 I/O 端口。 | **不需要专门的 I/O 指令**。<br>直接用 `MOV`、`LDR` 等普通访存指令读写端口。寻址灵活，但占用主存地址空间。 |
| **独立编址** | I/O-mapped I/O<br>(特权 I/O) | I/O 地址空间和内存地址空间**物理独立，各有一套编码**。<br>内存有 $0 \sim 4GB$，I/O 端口也有 $0 \sim 64KB$。 | **必须设置专属的 I/O 指令**。<br>如 x86 的 `IN` 和 `OUT` 指令。不占用主存空间，但指令单一、寻址不灵活。 |

### 三、 程序查询方式与 CPU 独占率计算（计算必拿分）

程序查询方式的核心是**“死等”**。CPU 必须在一段查询程序中打转，不断读取状态寄存器，直到状态位 $D=1$（准备就绪）。

```assembly
.L_POLLING:
    IN  AL, [STR_ADDR]   # 读取状态端口
    TEST AL, 0x01        # 测试就绪位是否为 1
    JZ  .L_POLLING       # 若不为 1 (准备中)，继续循环查询
    IN  AL, [DBR_ADDR]   # 若就绪，从数据缓冲端口读入数据
```

#### 🌰 408 经典计算题秒杀模板
**【典型真题预测】**：
某系统 CPU 主频为 $500\text{MHz}$。某外设采用**程序查询方式**进行输入，数据传输率为 $50\text{KB/s}$（以字节为单位传输）。每次查询并传输 1 字节的数据需要执行 $10$ 条指令。这些指令的平均 $CPI = 2$。
求：该外设进行数据传输时，CPU 用于该外设服务的时间占总时间的比例是多少？

*   **三步秒杀套路法**：
    1.  **第一步：算外设的数据传输间隔（每隔多久传一次数据？）**
        外设传输率为 $50\text{KB/s} = 5 \times 10^4 \text{ Byte/s}$。
        则每传输 1 字节，时间间隔为：
        $$T_{\text{interval}} = \frac{1}{5 \times 10^4} \text{ s} = 20\mu\text{s}$$
    2.  **第二步：算每次服务占用 CPU 的时钟周期数（花多少周期？）**
        每次传输服务执行 10 条指令，平均 $CPI = 2$，则共耗费时钟周期：
        $$N_{\text{cycle}} = 10 \times 2 = 20 \text{ 个时钟周期}$$
        因为 CPU 主频为 $500\text{MHz}$（时钟周期为 $T = \frac{1}{500\text{MHz}} = 2\text{ns}$），这些周期实际物理耗时为：
        $$T_{\text{service}} = 20 \times 2\text{ns} = 40\text{ns}$$
    3.  **第三步：算 CPU 时间占用比例（占百分之几？）**
        在每 $20\mu\text{s}$ 的时间间隔内，CPU 要花 $40\text{ns}$ 的时间去伺候这个外设。
        $$\text{占用比例} = \frac{T_{\text{service}}}{T_{\text{interval}}} = \frac{40\text{ns}}{20\mu\text{s}} = \frac{40 \times 10^{-9}\text{s}}{20 \times 10^{-6}\text{s}} = 0.2\%$$
