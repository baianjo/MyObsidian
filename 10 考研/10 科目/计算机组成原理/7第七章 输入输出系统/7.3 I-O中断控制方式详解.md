### 一、 中断处理流程：硬件与软件的完美交接（必背大题点！）

当中断源发出请求到 CPU 完成处理，整个过程被一条无形的红线分割为**硬件自动完成（中断隐指令）**和**软件执行指令（中断服务程序）**两个阶段。

```
[ 当前指令结束 ] ➜ 🚨 检测到中断信号
       │
   ( 阶段 1: 硬件自动完成 - 中断隐指令 )
       ├─> 1. 关中断 (EINT = 0)
       ├─> 2. 保存断点 (PC 和 PSW 压栈)
       └─> 3. 引出中断服务程序 (根据向量表修改 PC)
       │
   ( 阶段 2: 软件指令执行 - 中断服务程序 )
       ├─> 4. 保护现场 (PUSH 通用寄存器)
       ├─> 5. 执行实际 I/O 服务
       ├─> 6. 恢复现场 (POP 通用寄存器)
       ├─> 7. 开中断 (EINT = 1)
       └─> 8. 中断返回 (IRET 指令)
```

#### 1. 硬件响应：中断隐指令（非真实指令！）
“中断隐指令”是计算机里的一种**硬件逻辑结构**，不是汇编指令，CPU 硬件在检测到中断后自动完成以下三步：
1.  **关中断**：将允许中断触发器（EINT）置 0，防止新来的中断打乱当前断点的保存过程。
2.  **保存断点**：把当前的 $PC$（下一条指令的地址）和 $PSW$ 强行压入系统堆栈。
3.  **引出中断服务程序**：根据中断源寻找对应的入口地址写入 $PC$。

> [!danger] 👑 985 概念高频坑：向量地址 vs 入口地址
> 考研极喜欢出单选题考这两者的区别，必须秒杀：
> *   **中断向量**：指**中断服务程序的入口地址**。
> *   **中断向量表**：存放所有中断向量的内存区域。
> *   **向量地址**：指**中断向量表某一项的内存物理地址（即存放入口地址的地址）**。
> *   **逻辑链条**：**中断源 ➜ 产生向量地址 ➜ 读向量地址处的内存 ➜ 读出入口地址（中断向量） ➜ 送入 PC**。

### 二、 单重中断 vs 多重中断

*   **单重中断（单级中断）**：在执行中断服务程序时，不允许被任何新中断打断。全程处于**关中断**状态。
*   **多重中断（嵌套中断）**：在执行低优先级中断时，允许被高优先级的中断源再次打断，形成类似嵌套函数的结构。
    *   **实现嵌套的必要条件**：在中断服务程序一开始**保护完现场后，必须立即执行“开中断”指令**！

### 三、 屏蔽字与多重中断优先级推演（408 大题必考👑）

每个中断源都有一个**中断屏蔽寄存器**，存放一个**屏蔽字（Mask Word）**：
*   **物理意义**：屏蔽字中的某一位为 **1** 代表**屏蔽（不响应）**对应设备的中断；为 **0** 代表**不屏蔽（允许响应）**。

#### 1. 屏蔽字书写三大黄金铁律
1.  **自己必须屏蔽自己**：屏蔽字中，自己对应的位永远是 **1**。
2.  **高优先级屏蔽低优先级**：优先级越高的设备，其屏蔽字里 **1** 的个数越多。
3.  **终极推导公式**：如果设备 $A$ 的优先级高于设备 $B$，那么设备 $A$ 的屏蔽字中，对应 $B$ 的那一项必须是 **1**（代表 $A$ 运行时，屏蔽 $B$ 的干扰）。

#### 2. 大题经典推推乐
**【例题】**：系统有 A、B、C、D 四个中断源，其硬件排队优先级为 $A > B > C > D$。现在我们通过设置屏蔽字，把它们的**实际处理优先级**调整为：**$D > A > C > B$**。
请写出每个中断源的屏蔽字。

*   **秒杀步骤**：
    1.  横纵列出表格。
    2.  对于每一行设备，去对照它的**实际处理优先级**。
    3.  **如果目标设备的实际优先级 $\le$ 当前设备，该位置 1；否则置 0**。
        *   例如：对于 D（实际优先级最高），它能屏蔽所有设备（包括自己），所以全为 1。
        *   对于 A（实际优先级第二），它能屏蔽 A、C、B，但不能屏蔽 D，所以 D 位为 0，其余为 1。

| 中断源 | A | B | C | D | 屏蔽字 (按A-B-C-D顺序) |
| :---: | :---: | :---: | :---: | :---: | :--- |
| **A** | **1** | **1** | **1** | 0 | `1110` |
| **B** | 0 | **1** | 0 | 0 | `0100` |
| **C** | 0 | **1** | **1** | 0 | `0110` |
| **D** | **1** | **1** | **1** | **1** | `1111` |

### 🚨 避坑警告：关中断与开中断的时序
在多重中断的服务程序中，很多同学会漏掉**恢复现场前要关中断**的细节。
多重中断服务程序的正确软件逻辑是：
$$\text{保护现场} \to \text{置新屏蔽字} \to \mathbf{开中断} \to \text{执行服务} \to \mathbf{关中断} \to \text{恢复现场} \to \text{恢复旧屏蔽字} \to \text{中断返回}$$
为什么恢复现场前要关中断？因为如果此时不关中断，在 `POP` 恢复通用寄存器的中途被别的中断打断，会导致寄存器状态被彻底撕裂损毁！
