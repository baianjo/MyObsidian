### 一、 x86 架构通用寄存器体系 (32位 IA-32)

x86 的 8 个 32 位通用寄存器有着明确的物理分工，做题时要对其“曾用名”和“专职”了如指掌：

```
31                                15                7        0
+---------------------------------+-----------------+--------+
|                                 |       AH        |   AL   | -> EAX (Accumulator: 累加器, 存返回值)
+---------------------------------+-----------------+--------+
|                                 |       BH        |   BL   | -> EBX (Base: 基地址寄存器)
+---------------------------------+-----------------+--------+
|                                 |       CH        |   CL   | -> ECX (Counter: 计数器, 循环次数)
+---------------------------------+-----------------+--------+
|                                 |       DH        |   DL   | -> EDX (Data: 数据寄存器, 乘除法辅助)
+---------------------------------+-----------------+--------+
|                                 |                 SI       | -> ESI (Source Index: 源变址)
+---------------------------------+-----------------+--------+
|                                 |                 DI       | -> EDI (Destination Index: 目的变址)
+---------------------------------+-----------------+--------+
|                                 |                 BP       | -> EBP (Base Pointer: 栈帧基指针)
+---------------------------------+-----------------+--------+
|                                 |                 SP       | -> ESP (Stack Pointer: 栈顶指针)
+---------------------------------+-----------------+--------+
```

*   **EAX, EBX, ECX, EDX**：低 16 位可作为 **AX, BX, CX, DX** 独立访问；AX 的高低 8 位又可作为 **AH, AL** 访问。
*   **ESP**：栈顶指针，永远指向当前执行栈的栈顶。
*   **EBP**：栈底指针，用于定位当前函数栈帧内的局部变量和参数。

### 二、 Intel 格式 vs AT&T 格式 (408 核心选择题点)

408 真题中，两种格式交替出现。搞混方向直接扣分！

| 特征 | Intel 格式 (Windows 常用) | AT&T 格式 (Linux/GCC 常用) |
| :--- | :--- | :--- |
| **操作数顺序** | **`Op Dest, Src`** (写给读) | **`Op Src, Dest`** (读给写) |
| **寄存器名前缀** | 无前缀 (如 `eax`) | 前缀 **`%`** (如 `%eax`) |
| **立即数前缀** | 无前缀 (如 `5`, `0x40`) | 前缀 **`$`** (如 `$5`, `$0x40`) |
| **指令长度后缀** | 无后缀。用 `byte/word/dword ptr` 指明大小 | 有后缀：`b`(8位), `w`(16位), `l`(32位), `q`(64位) |
| **内存寻址表示** | `[base + index*scale + disp]` | `disp(base, index, scale)` |

#### 🌰 格式对比实战演练：
将常数 $8$ 加到寄存器 $EAX$ 指向的内存地址上：
*   **Intel 格式**：`add dword ptr [eax], 8`
*   **AT&T 格式**：`addl $8, (%eax)`

### 三、 常用 x86 汇编指令集详解

#### 1. 数据传送指令
*   `mov`：复制数据。`mov eax, ebx` (Intel) / `movl %ebx, %eax` (AT&T)。
*   `push`：压栈。先 $(ESP) - 4 \to ESP$，再将操作数写入 $ESP$ 指向的内存。
*   `pop`：出栈。先从 $ESP$ 处读出数据，再 $(ESP) + 4 \to ESP$。

#### 2. 算术运算指令
*   `add`/`sub`：加/减法。
*   `inc`/`dec`：自增/自减 1。
*   `mul`/`imul`（无符号 / 带符号乘法）：
    *   **单操作数格式**：`mul ecx`。被乘数默认在 **`EAX`** 中，计算 `EAX * ECX`，结果的 64 位存放在 **`EDX:EAX`**（EDX 存高 32 位，EAX 存低 32 位）。
    *   **双操作数格式**（仅 `imul` 支持）：`imul eax, ebx`。计算 `EAX * EBX`，结果截断存入 `EAX`。
*   `div`/`idiv`（无符号 / 带符号除法）：
    *   **单操作数格式**：`div ecx`。被除数默认在 **`EDX:EAX`** 中（64位），除以 `ECX`。
    *   **结果存法**：商存入 **`EAX`**，余数存入 **`EDX`**。

> [!danger] 避坑警告：乘除法的隐式寄存器改写！
> 408 大题中如果考到汇编分析，一旦看到 `div` 或 `mul`，**绝对要警惕 EDX 寄存器的内容被悄悄改写了**！
> 在除法前，必须用 `cltd`（或 `cdq`）指令将 `EAX` 中的有符号数符号扩展到 `EDX` 中（凑够 64 位被除数 EDX:EAX），否则除法会发生溢出或致命错误。

#### 3. 逻辑运算与移位指令
*   `and`/`or`/`xor`/`not`：按位与/或/异或/取反。
*   `shl`/`shr`（逻辑左/右移）：高位/低位补 0。
*   `sar`（算术右移）：**高位补符号位**（常用于除以 2 的幂次）。

### 四、 复杂的内存引用表示法（必考物理模型）

在 x86 中，内存寻址公式为：
$$\text{有效地址 } EA = \text{Base} + (\text{Index} \times \text{Scale}) + \text{Displacement}$$
*   **Base**（基址寄存器）：任何通用寄存器。
*   **Index**（变址寄存器）：除 ESP 外的任何通用寄存器。
*   **Scale**（比例因子）：只能是 **1, 2, 4, 8**。对应 `sizeof(char/short/int/double)`。
*   **Displacement**（偏移量）：立即数。

#### 🌰 C 语言数组寻址的汇编翻译：
假设 `int arr[100]`，`arr` 首地址在 `%edx` 中，变量 `i` 在 `%ecx` 中。要访问 `arr[i]` 并存入 `%eax`：
*   **AT&T 汇编**：`movl (%edx, %ecx, 4), %eax`
*   **计算过程**：$EA = (\%edx) + (\%ecx) \times 4$。比例因子为 4，因为 `int` 占 4 字节。
*   **Intel 汇编**：`mov eax, [edx + ecx*4]`
