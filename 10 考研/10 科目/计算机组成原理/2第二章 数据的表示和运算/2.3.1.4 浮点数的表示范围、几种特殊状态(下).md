## 🌟 考研核心黄金表：IEEE 754 状态全景图 (直接背诵)

任何 IEEE 754 浮点数，只需看它的 **阶码 (Exponent)** 和 **尾数 (Mantissa)**，就能瞬间对号入座：

| 阶码 (E) | 尾数 (M) | 代表什么状态？ | 隐含的最高位 | 阶码真值解读 (Float型) |
| :--- | :--- | :--- | :---: | :--- |
| **全 0** | **全 0** | **真值 0 ($\pm0$)** | 0 | - |
| **全 0** | **不全为 0** | **非规格化数** (逼近0的极小数) | 0 | **固定为 -126** ==(极易错)== |
| **1 ~ 254** | **任意** | **规格化数** (正常数) | 1 | E - 127 |
| **全 1** | **全 0** | **无穷大 ($\pm\infty$)** | - | - |
| **全 1** | **不全为 0** | **NaN** (非数，Not a Number) | - | - |

## 1️⃣ 可视化：当一个浮点数“越来越小”会发生什么？

设想你在不断把一个正浮点数除以 2，它的值在数轴上向 0 逼近。

`[规格化数最小正值]` ➡️ `[非规格化数区间]` ➡️ `[真值 0]`

1. **正下溢 / 负下溢 (Underflow)：** 只要运算结果落在规格化数能表示的**最小正数**和**最大负数**之间（即绝对值太小了，规格化数装不下了），统称为**下溢**。
2. **硬件层面的“下溢异常 (UE flag)” (⚠️408爱考细节)：**
   * 下溢到 **非规格化数** 区间 ➡️ **不报错**，正常存（默认不响应异常，程序继续跑）。
   * 下溢到 **机器 0** 区间 ➡️ **硬件置下溢异常标志位为1**（但754标准依然默认不终止程序，除非程序员手动接管）。

> **真值 0 怎么存？**
> 阶码、尾数必须**全为0**。
> Float 正零：`0 00000000 000...0`
> Float 负零：`1 00000000 000...0` (仅符号位不同，逻辑上等价)

## 2️⃣ 抢分深水区：非规格化数 (Denormalized)

非规格化数的作用是**填补“最小规格化正数”到“0”之间的空白**。它有**两处与常规数字截然不同的特殊解读规则**，出卷老师最爱在这里挖坑：

### 🛑 必考大坑 1：隐含位变成了 `0`
* **规格化数：** 尾数前面隐含 `1.` (例如 $1.M \times 2^E$)
* **非规格化数：** 尾数前面隐含 `0.` (例如 $0.M \times 2^E$)，这样才能表示更小的值！

### 🛑 必考大坑 2：阶码真值是固定的！不是减出来的！
* 此时阶码的机器数是 **全0**。
* **错误算法 (必死)：** 按照无符号数算 $0 - 127 = -127$ ❌
* **标准规定 (保命)：** 非规格化数的阶码真值**固定解读为 $1 - Bias$**。
  * Float型 偏置值 $Bias = 127$，所以阶码真值**永远是 $1 - 127 = -126$** ✅
  * Double型 偏置值 $Bias = 1023$，所以阶码真值**永远是 $1 - 1023 = -1022$** ✅

### 🎯 满分推导：Float型的边界值 (STT STT STT 考点)
*前提：符号位为0 (正数)，阶码为全0 (真值-126)，尾数隐含0.*

*   **最小正值 (尽量让尾数最小)：**
    *   尾数不能全0 (全0就是机器0了)，所以只能是最后一位为1。
    *   尾数 $M = 0.00...001_{(2)} = 2^{-23}$
    *   **最终值：** $2^{-23} \times 2^{-126} =$ **$2^{-149}$**
*   **最大正值 (尽量让尾数最大)：**
    *   尾数 23位全为1。
    *   尾数 $M = 0.11...111_{(2)} = 1 - 2^{-23}$
    *   **最终值：** $(1 - 2^{-23}) \times 2^{-126} =$ **$2^{-126} - 2^{-149}$**

*(注：非规格化负数同理，加个负号即可。)*

## 3️⃣ 极易混淆的特殊状态：NaN vs Infinity (无穷)

### 状态特征：
只要 **阶码全为 1**，它就不再是一个常规数字了！
*   **无穷 (Inf)：** 尾数 **全为0**。(符号位0为正无穷，1为负无穷)
*   **非数 (NaN)：** 尾数 **不全为0**。(符号位是什么无所谓)

### 💥 出题老师的“变态”陷阱 (STT 压轴题常见)：
程序中遇到除以 0 怎么办？IEEE 754 容错性极高，**不会报错中断**，而是会产生特殊值！
*   `0 / 0` = **NaN**
*   负数开根号 ($\sqrt{-1}$) = **NaN**
*   $\infty - \infty$ = **NaN**
*   `非零实数 / 0` (例如 `1.0 / 0`) = **正无穷 (+Inf)** *(STT 这里极易和 0/0 混淆！STT)*
*   `-1.0 / 0` = **负无穷 (-Inf)**

## 4️⃣ 真题实战演示 (2023年408第14题复现)

**已知：** 某 Float 型机器数为 `0x8040 0000`，求其真值。

**学长带你3秒拆解：**
1. **转二进制拆分：**
   `0x8040 0000` = `1_00000000_1000...0`
   * 符号位：`1` (负数)
   * 阶码：`0000 0000` (**全0！警觉！触发非规格化数规则！**)
   * 尾数：`100 0000...0` (**不全为0**)
2. **应用非规格化规则 (套模板)：**
   * 阶码真值绝对不能减 $0-127$，而是固定死：**$-126$**
   * 尾数小数点前补 `0.`，即 $0.100...0_{(2)}$，转换成十进制是 $0.5$ (即 $2^{-1}$)
3. **组合计算：**
   真值 = $- (0.5 \times 2^{-126}) =$ **$-2^{-127}$** (这就是答案，绝对拿满分)

## 5️⃣ 终极拓展：Double 双精度型边界值 (防超纲必备)

*为了达成985的高要求，STT留下的作业我帮你全部推演完毕，直接记下来应对冷门大题。*

**Double 型基本盘：** 1位符号 + **11位阶码** + **52位尾数**。偏置值 = $1023$。

| 状态类型 | Float型 边界值 (常考) | Double型 边界值 (防超纲) |
| :--- | :--- | :--- |
| **规格化数 最小正值**<br>*(阶码真值=1-Bias, 尾数全是0)* | $1.0 \times 2^{-126}$ = **$2^{-126}$** | $1.0 \times 2^{-1022}$ = **$2^{-1022}$** |
| **规格化数 最大正值**<br>*(阶码=全1-1, 尾数全1)* | $(2 - 2^{-23}) \times 2^{127}$ | $(2 - 2^{-52}) \times 2^{1023}$ |
| **非规格化数 最小正值**<br>*(阶码固定，尾数最后一位为1)* | $2^{-23} \times 2^{-126}$ = **$2^{-149}$** | $2^{-52} \times 2^{-1022}$ = **$2^{-1074}$** |
| **非规格化数 最大正值**<br>*(阶码固定，尾数全1)* | $(1 - 2^{-23}) \times 2^{-126}$ | $(1 - 2^{-52}) \times 2^{-1022}$ |

**Double 的机器 0 与特殊状态：**
*   正零：`0_00000000000_全0`
*   负零：`1_00000000000_全0`
*   正无穷 (+Inf)：`0_11111111111_全0`
*   负无穷 (-Inf)：`1_11111111111_全0`
*   NaN：`符号位任意_11111111111_不全为0`
