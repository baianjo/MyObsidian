# 🚀 核心应试密码：IEEE 754 浮点数体系
> **考情警告**：几乎每年必考！主要考察**十进制与机器数互转**、**复杂小数处理**、**机器数多重解读(int与float的辨析)**。必须做到绝对熟练，这里的1分都不允许丢。

## 🧠 一、 必须刻在DNA里的死记硬背模板

不论是Float还是Double，内部结构永远是：**符号位(S) + 阶码(E) + 尾数(M)**

| 类型 | 总位数 | 结构拆分 (S + E + M) | 阶码(E)偏置值 |
| :--- | :--- | :--- | :--- |
| **单精度 (Float)** | 32 bits | **1** + **8** + **23** | **127** ($2^{8-1}-1$) |
| **双精度 (Double)** | 64 bits | **1** + **11** + **52** | **1023** ($2^{11-1}-1$) |

🚨 **两个致命考点（极易掉坑，考试必设陷阱）：**
1. **阶码(E)是移码表示**：移码真值计算 = 无符号机器数 - 偏置值。
2. **尾数(M)隐含最高位的 `1.`**：存入机器时**必须丢弃**最高位的1，取出计算时**必须补上**这个 `1.`。

---

## ⚔️ 二、 题型一：十进制真值 $\rightarrow$ IEEE 754 浮点数

**【通杀算法步骤】**
1. **转二进制**：整数部分除2取余，小数部分乘2取整。
2. **科学计数法**：写成 $\pm 1.M \times 2^E$ （规格化，小数点前必须只有1个1）。
3. **拼装三段论**：
   - **S (符号)**：正为0，负为1。
   - **E (阶码)**：真实指数 $E$ + 偏置值(127或1023) $\rightarrow$ 转无符号二进制。
   - **M (尾数)**：**去掉**开头的 `1.`，直接填入，不够的位数在后面**补0**。

### 📌 满分演示示例：将 $-8.25$ 存为 Float
1. 转二进制：$-8.25_{(10)} \rightarrow -1000.01_{(2)}$
2. 科学计数法：$-1.00001 \times 2^3$
3. 拼装：
   - **S** = `1` （负数）
   - **E** = 真值 $3 + 127 = 130$ $\rightarrow$ `10000010`
   - **M** = 丢弃 `1.`，剩下 `00001`，补齐23位 $\rightarrow$ `00001 000000000000000000`
4. **最终机器数**：`1 10000010 00001000000000000000000` (十六进制记为 `0xC1040000`)

---

### 💡 提速外挂：复杂小数的“乘积取整法”
**适用场景**：遇到如 `0.4375` 这种无法一眼看出二进制的恶心小数。
**操作口诀**：**乘基数(2) $\rightarrow$ 整数部分拿走当比特(先出高位) $\rightarrow$ 剩余小数部分继续乘**，直到小数为0。

**实战 `0.4375` 转二进制：**
- $0.4375 \times 2 = 0.875$  $\rightarrow$ 取整数 **`0`** (小数点后第1位)
- $0.875 \times 2 = 1.75$    $\rightarrow$ 取整数 **`1`** (小数点后第2位)
- $0.75 \times 2 = 1.5$      $\rightarrow$ 取整数 **`1`** (小数点后第3位)
- $0.5 \times 2 = 1.0$       $\rightarrow$ 取整数 **`1`** (小数点后第4位)，小数清零，结束！
> **结论**：$0.4375_{(10)} = 0.0111_{(2)}$

## 🛡️ 三、 题型二：IEEE 754 机器数 $\rightarrow$ 十进制真值

**【逆向破解步骤】**
1. 把十六进制展开为二进制，按 **1-8-23** 强行切断。
2. 算符号：0正1负。
3. 算阶码：8位二进制当成**无符号数**算十进制 $\rightarrow$ 再**减去127** = 真实指数 $E$。
4. 算尾数：强行在前面拼上 **`1.`**。
5. 组装化简：$(-1)^S \times (1.M) \times 2^E$。

### 📌 满分演示示例（2013年真题）
> 给出十六进制 Float，问真值？(过程略，直击核心转换)
- 假设切分出 S = `1`，E = `10001100`，M = `1000...0`
- **E解码**：`10001100` 无符号十进制是 $140$。真指数 = $140 - 127 = 13$。
- **M解码**：尾数串是 `100...`，补上隐含的 `1.` $\rightarrow$ 二进制为 `1.1` $\rightarrow$ 十进制为 $1.5$。
- **最终计算**：$-1.5 \times 2^{13}$。

## 🔮 四、 题型三：32位机器数的多重解读 (Float vs Int 补码)

**【考场破题思维（2020年真题杀招）】**
遇到给了一串 32 bit 机器数问真值的题，如果没有明确说是 Float：
1. 先按 Float 规则算一遍，看选项有没有（通常选项要么全有要么全无）。
2. 如果按 Float 算出来答案对不上，**它就是带符号整数 (int)**，按照**补码**规则去算！

### ⚡ 提速外挂：补码光速转原码（一眼看出十进制）
**常规方法**：减1取反，容易算错。
**考研必秒法（从右向左扫描法）**：
1. 观察补码序列，**从右往左**找，找到**第一个比特 `1`**。
2. 划一条线：这个 `1` 和它**右边**的所有位 **保持绝对不变**。
3. 线的**左边**（不包括最高符号位）：**0变1，1变0** 全部翻转！
4. 符号位保留，此时直接变成原码，光速算出十进制。

**实战示例**：找补码 `1010` 的原码
1. 从右往左找到第一个 `1`，在第2位：`10 | 10`
2. 右边 `10` 不变。
3. 左边 `0` 翻转为 `1`。符号位最高位 `1` 不变。
4. 结果为 `1110` (即符号为负，数值为 110，真值为 -6)。绝对不会错，不掉一分！
