# 🧠 定点数的加减运算与溢出判断

> **💡 985考研复习指导：**
> 本节**极其重要**，是后续乘除法和ALU设计的基础。
> **原码加减法**硬件实现太复杂，考试极少考硬件逻辑，会笔算即可；
> **重点死磕：** 补码的加减运算逻辑，以及**三种溢出判断方法**（408选择题/大题必考核心）。

---

## 一、 核心运算规则：为什么一定要用补码？

原码加减法在硬件上非常恶心（需要判断符号位、比较绝对值大小），所以**计算机内部的加减法一律用补码实现**，直接将符号位当成数值位一起算，减法统一转为加法。

### 1. 补码加减法通用公式（一招吃遍天下）
无论加减，直接扔进加法器：
*   **加法：** $[A+B]_补 = [A]_补 + [B]_补$
*   **减法：** $[A-B]_补 = [A]_补 + [-B]_补$

### 2. ⚡ 考场秒杀技巧：已知 $[B]_补$ ，如何光速求 $[-B]_补$ ？
不要去傻傻地转回原码再转补码，直接用以下两种方法：
*   **方法一（常规）：** 连同符号位在内，**全部取反，末位 + 1**。
*   **方法二（肉眼秒看，极推）：** 从右往左找第一个 `1`。**这个 `1` 及其右边保持不变，这个 `1` 左边的所有位（含符号位）全部取反**。
    > **演示示例：**
    > 假设 $[B]_补$ = `1101 1000`
    > 从右往左找第一个1：`1101 1` **|** `000`
    > 左边取反，右边不变：`0010 1` **|** `000`  👉 这就是 $[-B]_补$ ！

## 二、 溢出（Overflow）：到底什么是溢出？

**概念：** 运算结果超出了该数据位数所能表示的范围。（比如8位补码只能表示 -128 ~ 127，你算出个 139，就溢出了）。

**⚠️ 绝对铁律（判断大前提）：**
1.  **同号相加**才有可能溢出！
2.  **异号相加**（一正一负）**绝对不可能溢出**！

| 溢出类型 | 发生场景 | 表现（考点） |
| :--- | :--- | :--- |
| **上溢 (正溢出)** | 正数 + 正数 | 结果变成了**负数** (超出了最大正数) |
| **下溢 (负溢出)** | 负数 + 负数 | 结果变成了**正数** (超出了最小负数) |

## 三、 👑 硬件判断溢出的3种方法（一分不丢核心区）

这是考试的重中之重，必须能够看懂逻辑表达式和电路图。

### 方法1：一位符号位逻辑表达式判断（看符号）
设 $A_S$ 为被加数符号位，$B_S$ 为加数符号位，$S_S$ 为结果符号位。
*   **逻辑表达式：** $V = A_S B_S \overline{S_S} + \overline{A_S} \overline{B_S} S_S$
*   **说人话翻译：**
    *   $A_S B_S \overline{S_S}$ ：负 + 负 = 正 （下溢）
    *   $\overline{A_S} \overline{B_S} S_S$ ：正 + 正 = 负 （上溢）
*   **判断标准：** $V = 1$ 时，说明发生溢出。

### 方法2：进位位判断（看进位 - 极其常考）
观察符号位的进位（$C_S$）和 最高数值位的进位（$C_1$）。
*   **计算公式：** 溢出标志 $V = C_S \oplus C_1$  *(异或运算：相同为0，不同为1)*
*   **判断标准：**
    *   $C_S \neq C_1$ (异或结果为1)：**溢出！**
    *   $C_S = C_1$ (异或结果为0)：没溢出。
*   **细分情况：**
    *   $C_S = 0, C_1 = 1$ 👉 正数+正数进位=负数 👉 **上溢**
    *   $C_S = 1, C_1 = 0$ 👉 负数+负数无进位=正数 👉 **下溢**

### 方法3：双符号位判断（模4补码）
将符号位扩展为两位。正数符号位为 `00`，负数符号位为 `11`。
*   **运算规则：** 两个符号位都参与加法运算。
*   **结果解读（高位是真符号，低位是算出来的假符号）：**
    *   `00`：结果为正，无溢出。
    *   `11`：结果为负，无溢出。
    *   `01`：本应为正，算出为负 👉 **上溢**。
    *   `10`：本应为负，算出为正 👉 **下溢**。
*   **硬件实现：** $V = S_{S1} \oplus S_{S2}$ (两个符号位异或，不同即为溢出)。

## 四、 🚨 985拔高 / 容易掉坑的边角料细节

1.  **模2补码 vs 模4补码：**
    *   **单符号位补码** = 模2补码（把符号位后面的逗号看作小数点，符号位的位权是 $2^0$，大于等于2的进位被硬件自然丢弃）。
    *   **双符号位补码** = 模4补码（高位符号位位权是 $2^1=2$，再往上进位的位权是 $2^2=4$，大于等于4的进位被丢弃）。
2.  **【高频避坑】双符号位占内存吗？**
    *   **绝不！** 在内存/主存中存储时，双符号位补码**依然只存1个符号位**（为了省空间）。
    *   只有在数据被送入ALU（算术逻辑单元）**准备进行运算的那一瞬间**，硬件才会自动复制一位符号位变成两个，运算完写回内存又会变回一个。
