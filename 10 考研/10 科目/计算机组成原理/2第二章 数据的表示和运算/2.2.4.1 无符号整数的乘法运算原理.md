# 【408必考】无符号整数的硬件乘法运算（不丢分指南）

> 💡 **专家批注**：
> 考研408对这块的考法非常固定，**绝对不会**让你手算简单的十进制乘法。
> **必考点只有三个**：硬件寄存器谁干嘛的、每过一个时钟周期各个寄存器里的值是多少（**表格追踪题**）、怎么判断溢出。
> 把下面这份笔记吃透，关于无符号乘法的选择题和大题，你一分都不会丢。

---

## 🚀 一、 核心硬件部件与“工位”分配
不要去管小学乘法怎么算，直接背硬件是怎么模拟这个过程的。考试常问“初始状态”和“最终结果”在哪。

假定是两个 $n$ 位无符号数相乘：
*   **寄存器 X**：存放**被乘数**（固定不动）。
*   **寄存器 Y**：存放**乘数**。一边算，一边右移，算完后存放乘积的**低 $n$ 位**。
*   **寄存器 P**：存放**部分积**。初始值为**全0**，算完后存放乘积的**高 $n$ 位**。
*   **触发器 C**：存放加法产生的**最高位进位**（只存1个比特）。
*   **计数器 Cn**：初始值为 $n$（比特位数），减到 0 运算结束。

---

## ⚙️ 二、 运算流程（做题的机械步骤）
408极爱考：*“经过第 $k$ 次循环后，寄存器 P 和 Y 的内容是什么？”*
记住这个 **“看位 $\rightarrow$ 加法 $\rightarrow$ 移位”** 的三步死循环：

1.  **看位**：看寄存器 Y 的最低位（记作 $Y_0$）。
    *   如果 $Y_0 = 1$，则让 ALU 执行加法：**`P = P + X`**，产生的进位存入 C。
    *   如果 $Y_0 = 0$，则**什么也不加**（或者理解为 `P = P + 0`，C = 0）。
2.  **移位（🌟超级大考点）**：
    *   将 **`{C, P, Y}`** 视为一个整体，**统一逻辑右移 1 位**！
    *   *说人话：整体往右挪，最右边的位直接丢进垃圾桶，最左边（C的高位）**补0**。*
3.  **打卡**：计数器 Cn 减 1。如果 Cn 不为 0，回到第1步；若为 0，结束。

> ⚠️ **【绝不丢分陷阱】** 
> 只要是**无符号数**相乘，移位**一定是【逻辑右移】（高位无脑补0）**！
> 千万别和有符号数的补码乘法（Booth算法，算术右移）搞混！这也是408最爱挖的坑！

## 📊 三、 考场无敌演示表格（以 1101 × 1011 为例）
> 被乘数 X = 1101 (13)
> 乘数 Y = 1011 (11)
> 位数 $n=4$，需循环 4 次。



| 步骤/轮次 | 动作详解 (说人话) | 进位 C | 部分积 P | 乘数 Y |
|---|---|---|---|---|
| 0. 初始状态 | P置全0，X不动，Y装入乘数。 | 0 | 0000 | 1011 |
| 1. 第一轮 | ① 看Y末位是 1，P = P + X (1101)，进位为0 | 0 | 1101 | 1011 |
|  | ② {C,P,Y} 整体逻辑右移1位，高位补0 | 0 | 0110 | 1101 |
| 2. 第二轮 | ① 看Y末位是 1，P = P + X (1101)，产生进位1 | 1 | 0011 | 1101 |
|  | ② {C,P,Y} 整体逻辑右移1位，高位补0 | 0 | 1001 | 1110 |
| 3. 第三轮 | ① 看Y末位是 0，不加 (P不变)，C为0 | 0 | 1001 | 1110 |
|  | ② {C,P,Y} 整体逻辑右移1位，高位补0 | 0 | 0100 | 1111 |
| 4. 第四轮 | ① 看Y末位是 1，P = P + X (1101)，产生进位1 | 1 | 0001 | 1111 |
|  | ② {C,P,Y} 整体逻辑右移1位，高位补0 | 0 | 1000 | 1111 |

**🎯 最终结果看哪里？**
循环结束，结果保存在 `{P, Y}` 拼接的 $2n$ 个比特中：`1000 1111` （等于十进制 143，计算完全正确）。原乘数Y的值早就被移位挤出去了，全部丢失！

## 💥 四、 乘法溢出的判断与处理（拿满分的最后一环）
两个 $n$ 位数相乘，理论上需要 $2n$ 位才能存下全集。但计算机（如C语言中的 `unsigned int`，32位乘32位，结果通常只保留低32位）常常会**强行截断，只保留低 $n$ 位（也就是 Y 寄存器里的值）**。

### 1. 怎么判断溢出？（秒杀考点）
*   **硬件判断条件**：直接看**高 $n$ 位（寄存器 P 里的值）**。
*   **判断逻辑**：如果寄存器 P 里的值 **不全为 0**，说明高位有有效数据被丢弃了 $\Rightarrow$ **发生溢出！此时硬件会将溢出标志位  CF 置为 1**。
    *(如上方演示表格最终 P=1000 不全为0，若只保留低4位 1111，则必然溢出)*

### 2. 溢出了怎么办？（了解即可，防选择题冷门选项）
*   **方法一：直接摆烂（忽略）**。得到一个被截断的错误结果（C语言默认操作）。
*   **方法二：不摆烂（引发异常）**。在汇编代码的乘法指令后面，插一条**溢出自陷指令**（如 x86 的 `into` 指令）。
    *   *原理*：该指令专门盯着 `OF` 标志位。只要看到 `CF == 1`，立刻触发操作系统的**异常处理程序 (Trap)**，终止或者挽救错误。
