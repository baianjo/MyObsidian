# 浮点数加减运算：舍入（Rounding）

> **💡 核心考点与背景（为什么要做舍入？）**
> 浮点数加减运算分五步：1.对接 -> 2.尾数加减 -> 3.规格化 -> **4.舍入** -> 5.溢出判断。
> 在**对接**和**尾数加减**后，尾数的位数可能会超过寄存器能保存的极限（对于单精度float，隐含1位+23位尾数=24位）。硬件通常会额外保留3位冗余位（保护位、舍入位、粘滞位）。**舍入的目的，就是把这多出来的3位砍掉，并决定要不要向前进位（+1）。**

---

## 一、 考研绝对重点：就近舍入 (Round to Nearest)
IEEE 754 标准规定了四种舍入模式，**默认且考试必考的是“就近舍入”**（2017年统考43题考点）。
*注：截断法（向零舍入）、正向舍入、负向舍入只需知晓名字，底层规则不深究（必不考计算）。C语言修改舍入模式的函数调用必不考，直接跳过。*

### 🧠 核心心法：类似于“0舍1入”，外加“偶数法则”
你可以把它类比为十进制的“四舍五入”，但在二进制里，我们看的是被舍弃掉的**最高位**是不是1。

为了绝对不丢分，请严格按照以下**优先级树**进行判断：
设我们要把多余的位数砍掉，此时视线聚焦在砍刀切开的地方 ✂️：
**[保留位的最低位]** ✂️ **[被舍弃的最高位]** [被舍弃的其余位]

*   **规则 1：无脑舍弃（0舍）**
    *   如果 **[被舍弃的最高位]** 是 `0` 👉 直接全部砍掉，前面不动。
*   **规则 2：无脑进位（1入 - 普通情况）**
    *   如果 **[被舍弃的最高位]** 是 `1`，且 **[被舍弃的其余位]** **不全为0**（如101, 110）👉 砍掉后，给 **[保留位的最低位]** 加1。
*   **规则 3：神仙打架，强制偶数（1入 - 刚好一半的极端情况）**
    *   如果被舍弃的位**刚好是** `100`（即刚好等于一半）👉 此时进不进位，取决于 **[保留位的最低位]**：
        *   如果保留位最低位是 `0` 👉 **不进位**（直接砍掉）。
        *   如果保留位最低位是 `1` 👉 **进位**（加1，使得最低位变成0）。
        *   *总结：当刚好卡在中间时，强行让最后保留的那一位变成 `0`（这叫凑偶数）。*

## 二、 极限硬核可视化：四个C语言例子的底层剖析

> **🔥 985专属防坑预警（极其重要）：**
> 在下面的例子中，被加数 $x = 2^{24}$（真值16777216）。
> 单精度Float的尾数是24位（1位隐含+23位显式）。此时小数点其实在第24位的后面！
> 此时，**保留位的最低位**，它的权重刚好是 $2^{-23} \times 2^{24} = 2^1 = 2$。
> **这意味着：只要舍入触发了“进位(+1)”，真值就会直接 $+2$！** 理解这一点，你就能彻底看懂下面的十进制结果为什么会跳跃。

### 🌰 例1：“0舍”
*   **代码**：`z = 16777216.0 + 0.5`
*   **二进制对齐相加后**：
    保留位: `1.000...000` ✂️ 舍弃位: `0xx` (0.5右移太多了，最高位变成了0)
*   **判定**：舍弃位最高位是0。触发**规则1**，直接截断。
*   **结果**：`z = 16777216.0` （0.5被完全吃掉了，精度丢失）。

### 🌰 例2：“1入” (非刚好一半)
*   **代码**：`z = 16777216.0 + 1.5`
*   **二进制对齐相加后**：
    保留位: `1.000...000` ✂️ 舍弃位: `1xx` (且后两位不全为0)
*   **判定**：舍弃位最高位是1，且不是刚好`100`。触发**规则2**，强制进位。
*   **结果**：保留位末位+1。由于末位权重是2，所以真值增加了2。
    打印结果：`z = 16777218.0`（本该是217.5，变成了218）。

### 🌰 例3：“刚好一半” —— 凑偶数（保留位末位本身是0）
*   **代码**：`z = 16777216.0 + 1.0`
*   **二进制对齐相加后**：
    保留位: `1.000...000` ✂️ 舍弃位: `100` （1.0右移后刚好变成100）
*   **判定**：舍弃位刚好是`100`，看向保留位末位。保留位末位是 `0`。触发**规则3**，为了凑偶数（保持末位为0），直接截断，**不进位**。
*   **结果**：`z = 16777216.0` （加了1等于没加）。

### 🌰 例4：“刚好一半” —— 凑偶数（保留位末位本身是1）
*   **代码**：设 $x = 16777218.0$（上文例2的结果，末位已经是1了）
    `z = 16777218.0 + 1.0`
*   **二进制对齐相加后**：
    保留位: `1.000...001` ✂️ 舍弃位: `100`
*   **判定**：舍弃位刚好是`100`，看向保留位末位。保留位末位是 `1`。触发**规则3**，为了凑偶数，**向前进位（+1）**，使得末位变成`1+1=10`（末位变成0了）。
*   **结果**：保留位末位+1，相当于真值又加了2。
    打印结果：`z = 16777220.0`（本该是219，变成了220）。

## 🎯 极速复习总结清单（背诵版）

考试遇到浮点数相加后尾数过长，需要舍去多余位（通常是算出来的一串二进制）：
1. 盯住**被踢掉的第一位**。
2. 是 `0`？👉 直接丢。
3. 是 `1`？盯住它后面的位。
   - 后面跟着 `1`（比如 `110`, `101`）？ 👉 直接给前面留下的末位加 `1`。
   - 后面全是 `0`（极端情况 `100...0`）？👉 看看前面留下的最后一位是啥。是 `1` 就加 `1`（让它变 `0`），是 `0` 就别动。目标就是**强迫末位变成0**。
