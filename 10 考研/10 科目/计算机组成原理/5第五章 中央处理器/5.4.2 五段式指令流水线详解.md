### 一、 五段流水线标准物理图谱

经典流水线分为五个标准阶段，每个阶段占用一个机器周期（即一个时钟周期）：

```
[ PC ] -> (IF: 取指) -> [IF/ID] -> (ID: 译码/读寄存器) -> [ID/EX] -> (EX: 执行/算地址) -> [EX/MEM] -> (MEM: 访存) -> [MEM/WB] -> (WB: 写回)
```

#### 1. 阶段 1：IF (Instruction Fetch, 取指)
*   **动作**：根据 PC 的值去指令 Cache（I-Cache）中取出指令，送入 **IF/ID 锁存器**。同时，PC 自动计算 $(PC) + 4 \to PC$。

#### 2. 阶段 2：ID (Instruction Decode, 译码 / 读寄存器)
*   **动作**：对指令进行译码，确定操作码。同时，**从通用寄存器堆中读出源操作数**送入 **ID/EX 锁存器**。

#### 3. 阶段 3：EX (Execute, 执行)
*   **动作**：ALU 登场进行实际计算。
    *   **算术逻辑指令**：进行对应的加减乘除或位运算。
    *   **Load/Store 指令**：计算内存的物理有效地址：$EA = \text{基址} + \text{偏移量}$。

#### 4. 阶段 4：MEM (Memory Access, 访存)
*   **动作**：访问数据 Cache（D-Cache）。
    *   **Load 指令**：根据 EX 段算出的地址，从 D-Cache 读出数据送入 **MEM/WB 锁存器**。
    *   **Store 指令**：将数据写入 D-Cache。
    *   **算术逻辑指令**：本阶段**无动作**，直接将结果透传到下一个锁存器。

#### 5. 阶段 5：WB (Write Back, 写回)
*   **动作**：把数据写回到通用寄存器堆中。
    *   **Load 指令**：把读出来的数写回目标寄存器。
    *   **算术逻辑指令**：把 ALU 的计算结果写回目标寄存器。
    *   **Store / 跳转指令**：本阶段**无动作**。

### 二、 流水线寄存器（锁存器）的物理功勋

*   **物理位置**：在每两个相邻阶段的物理交界处，都横亘着一个流水线寄存器：**IF/ID, ID/EX, EX/MEM, MEM/WB**。
*   **核心功能**：
    1.  **物理隔离**：保证前一阶段的微操作不会干扰下一阶段。例如，当 ID 阶段正在读取 R1 时，IF 阶段已经在读取下一条指令了，锁存器锁住了当前指令的译码信息，防止被冲刷。
    2.  **控制信号透传**：指令的所有控制信号在 ID 阶段一次性译码生成，随后**像传送带一样随着锁存器逐级往后传递**，直到其发挥作用的阶段。
*   **🚨 时钟周期计算公式（高频选择）**：
    $$\text{时钟周期 } \Delta t = \max(\text{各段物理延迟}) + \text{流水线寄存器（锁存器）延迟}$$

### 三、 核心指令在五段中的流向对比（大题命门！）

很多同学以为所有指令都要走完 5 段。其实对于某些指令，有些阶段是“空跑（透传）”的，这决定了写后读（RAW）冲突的间隔周期。

| 指令类型 | IF 段 | ID 段 | EX 段 | MEM 段 | WB 段 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **算术逻辑指令** | 取指令 | 译码/读寄存器 | **ALU 计算** | 空跑 (数据透传) | **写回目标寄存器** |
| **Load 指令** | 取指令 | 译码/读寄存器 | **算内存地址** | **读 D-Cache** | **写回目标寄存器** |
| **Store 指令** | 取指令 | 译码/读寄存器 | **算内存地址** | **写 D-Cache** | 空跑 (无动作) |
| **分支/跳转指令** | 取指令 | 译码/写 PC | 空跑/跳转判定 | 空跑 | 空跑 |

### 👑 985 高分大题秒杀技：分支跳转的“清空气泡数”分析

当流水线遇到跳转指令且确定跳转时，必须把已经错误取进流水线的后续指令“作废（清空）”。清空几个周期（几个气泡），取决于**跳转判定在哪个阶段完成**：

1.  **判定在 ID 阶段完成（最快）**：
    若转移指令在 ID 阶段算好了跳转目标地址，并直接修改了 PC。
    *   **时空分析**：当转移指令在 ID 阶段时，下一条指令已经在 IF 阶段被取出了。
    *   **代价**：只需清空这 1 条在 IF 段的错误指令（插入 **1 个气泡**）。
2.  **判定在 EX 阶段完成**：
    若跳转目标和跳转条件在 EX 阶段由 ALU 算完并修改 PC。
    *   **时空分析**：当转移指令在 EX 阶段时，第二条指令在 ID 阶段，第三条指令在 IF 阶段。
    *   **代价**：必须清空这两条指令（插入 **2 个气泡**）。
