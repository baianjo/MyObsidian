> [!abstract] 考点本质 (直击130分核心)
> 数据通路解决的核心问题是：**信息（数据/指令）如何以微操作的形式，在 CPU 内部的寄存器、ALU 和主存之间流动**。
> 408 核心考点：**单总线结构下的冲突解决机制（暂存器 Y 和 Z 的物理作用）、微操作控制信号（如 $PC_{out}$、$MAR_{in}$）的精准书写、以及专用数据通路（三总线/专用连线）大题的读图与秒杀**。

---

### 一、 CPU 内部单总线结构：冲突与解决

#### 1. 物理模型与致命缺陷
*   **物理特征**：所有的寄存器（PC, IR, MAR, MDR, ACC）和 ALU 的输入输出，全部连在一根**共享的 CPU 内部总线**上。
*   **致命缺陷**：**同一时刻，总线上只能有一个数据在传输**。
    *   例如，在计算 $A + B$ 时，ALU 需要同时输入操作数 $A$ 和 $B$。如果直接将它们连在单总线上，它们会瞬间在总线上碰撞混杂，变成无意义的乱码。

#### 2. 破局法宝：暂存寄存器 Y 与 Z
为了解决单总线的冲突，必须在 ALU 的入口和出口处引入**暂存寄存器 Y 和 Z**（对程序员完全透明/不可见）：
*   **Y 寄存器**：锁存操作数 $A$。
*   **Z 寄存器**：锁存 ALU 计算完的输出结果。

```
                     [ CPU 内部单总线 ]
                       |            |
                       v            v
                +------------+      |
                |  暂存器 Y   |      |
                +------------+      |
                       | (直连)     | (总线直连)
                       v            v
                     +----------------+
                     |    运算器 ALU  |
                     +----------------+
                             |
                             v
                       +------------+
                       |  暂存器 Z   |
                       +------------+
                             |
                             v
                     [ CPU 内部单总线 ]
```

### 二、 单总线结构微操作信号书写（高频手写题点）

我们以**取指周期**和**加法指令执行周期**为例，展示标准的微操作信号书写方式。

#### 1. 取指周期（FE）的微操作
1.  将 PC 送入 MAR，准备访存：
    $PC \to Bus \to MAR \implies \mathbf{PC_{out}, MAR_{in}}$
2.  发出读指令，同时 PC 自动加 1：
    $Mem(MAR) \to Bus \to MDR \text{ 且 } PC + 4 \to PC \implies \mathbf{Read, PC_{out}, Y_{in}, ALU_{add}, Z_{in}, Z_{out}, PC_{in}}$（408 中通常简化写为 $\mathbf{Read}$，同时 $PC$ 加 4 的控制信号由硬件自动触发）。
3.  把读取的指令送入 IR：
    $MDR \to Bus \to IR \implies \mathbf{MDR_{out}, IR_{in}}$

#### 2. 加法指令执行周期（EX）：`ADD (R0), R1`（将 R0 指向的内存数加到 R1）
1.  取出 R0 里的地址送往 MAR，准备读操作数：
    $R0 \to Bus \to MAR \implies \mathbf{R0_{out}, MAR_{in}, Read}$
2.  将 R1 的值送入暂存器 Y 锁存：
    $R1 \to Bus \to Y \implies \mathbf{R1_{out}, Y_{in}}$
3.  把读出来的内存数据（放在 MDR 中）送到总线上，与 Y 寄存器的数据一起送入 ALU 计算，结果送 Z 寄存器锁存：
    $MDR + Y \to Z \implies \mathbf{MDR_{out}, ALU_{add}, Z_{in}}$
4.  将计算结果 Z 送回 R1 寄存器：
    $Z \to Bus \to R1 \implies \mathbf{Z_{out}, R1_{in}}$

### 三、 专用数据通路（三总线/独立数据通路）

*   **物理特征**：寄存器和 ALU 之间不共享单一总线，而是拥有**多条内部总线**（如三总线：总线 A、总线 B、总线 C），或者直接在需要传输的部件之间拉专线。
*   **优点**：ALU 可以从两条总线（Bus A, Bus B）上同时读取两个操作数，结果直接通过 Bus C 输出，**完全不需要暂存器 Y 和 Z**，极大地提高了并行度。
*   **控制部件**：在专用通路中，通路方向的切换由**多路选择器（MUX）**或**三态门**控制。

### 👑 985 计组大题秒杀套路（408 核心大题破局法）

408 大题中如果给出一张复杂的专用数据通路图（通常在第五章的第一道大题），不要慌！请严格按以下步骤拆解：

#### 第一步：理清信息流方向（画箭头）
*   找到 **PC**、**IR**、**MAR**、**MDR**、**ALU**、**通用寄存器组**。
*   盯住数据通路的箭头，分清哪些是**输入（in）**，哪些是**输出（out）**。

#### 第二步：识别 MUX（多路选择器）的选择信号
*   如果 ALU 的 $A$ 端连着一个 $MUX$，它通常有多个输入源（如通用寄存器、PC、常数 4 等）。
*   控制信号如 $Mux_{sel} = 01$ 代表选择 PC 进 ALU。

#### 第三步：背诵“访存黄金微操作模版”
无论什么大题，只要涉及**读写内存**，底层动作必然是这几步：
*   **读内存（取指令或读操作数）**：
    $$\text{微操作： } (PC) \to MAR \implies \text{控制信号： } PC_{out}, MAR_{in}$$
    $$\text{微操作： } Mem(MAR) \to MDR \implies \text{控制信号： } Read, Mem_{in}, MDR_{in}$$
    $$\text{微操作： } MDR \to IR \implies \text{控制信号： } MDR_{out}, IR_{in}$$
*   **写内存（把数据存回内存）**：
    $$\text{微操作： } Address \to MAR \implies \text{控制信号： } Addr_{out}, MAR_{in}$$
    $$\text{微操作： } Data \to MDR \implies \text{控制信号： } Data_{out}, MDR_{in}$$
    $$\text{微操作： } MDR \to Mem(MAR) \implies \text{控制信号： } Write, Mem_{out}, MDR_{out}$$

#### 第四步：写控制信号时的“显式表示”
*   在写 408 答案时，如果是高电平有效，就直接写控制信号的名字，如 $\text{Read}, PC_{out}$。
*   如果图里标明了是低电平有效，记得加个非号 $\overline{Read}$！
