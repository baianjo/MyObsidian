## 一、 主存储器（内存）核心构成
> [!tip] 形象记忆：菜鸟驿站模型
> **MAR（地址）** = 取件码（找哪个位置）
> **存储体** = 货架（存放数据本身）
> **MDR（数据）** = 柜台（拿出来的包裹放这，或准备存的包裹放这）

### 1. 存储体层次结构（从大到小）
*   **存储体**：包含多个存储单元。
*   **存储单元**：存放一串二进制代码（一个**存储字 Word**）。
*   **存储元**：最小物理单元，由电容构成，只能存 **1 bit**（0或1）。

### 2. ==必考计算公式==：MAR 与 MDR
*   **MAR（存储器地址寄存器）**：指出要访问的存储单元地址。
    *   **核心考点**：`MAR的位数 反映 存储单元的总数`
    *   公式：**存储单元最大个数 = $2^{\text{MAR位数}}$**（例：MAR 4位 $\rightarrow$ 最多 $2^4=16$ 个存储单元）。
*   **MDR（存储器数据寄存器）**：暂存从存储体读出/写入的数据。
    *   **核心考点**：`MDR的位数 = 存储字长`
    *   意义：每次读写操作能处理的最大比特数。

> [!warning] 易混淆考点（字 vs 字节 vs 速率）
> 1. **字（Word）与 字节（Byte）**：
>    *   **1 字节 (1 Byte / 大B)** 永远 = **8 比特 (8 bit / 小b)**。
>    *   **1 个字 (Word)** 的大小不固定！取决于计算机设计（可能是16位、32位、64位）。MDR位数为16，则字长就是16位。
> 2. **大 B 与 小 b**：
>    *   网络带宽 `100M` 默认是 `100 Mbps`（每秒100M个**小b**）。
>    *   下载速度通常用 `MB/s`（**大B**），所以理论峰值是 $100 \div 8 = 12.5 \text{ MB/s}$。
> 3. **物理归属陷阱**：MAR和MDR**逻辑上**属于主存，但在现代计算机中，**物理上通常被集成在CPU内部**。

## 二、 CPU核心组件：运算器与控制器

### 1. 运算器（干苦力的：负责算术/逻辑运算）
| 部件简称 | 中文全称 | 核心作用及考点 |
| :--- | :--- | :--- |
| **ALU** | 算术逻辑单元 | **运算器的核心（造价最高）**，内部是复杂电路，实际执行加减乘除和逻辑运算的硬件。 |
| **ACC** | 累加器 | 存放操作数，**及运算的最终结果**（加减法的主力）。 |
| **MQ** | 乘商寄存器 | **专用于乘除法**。乘法时存乘数/结果低位；除法时存商。 |
| **X** | 通用寄存器 | 存放另一个操作数（暂存被乘数/加数等）。 |

### 2. 控制器（霸道总裁：负责指挥全场）
| 部件简称 | 中文全称 | 核心作用及考点 |
| :--- | :--- | :--- |
| **CU** | 控制单元 | **控制器的核心（造价最高）**，负责**分析指令**并发出各种控制信号。 |
| **IR** | 指令寄存器 | 存放**当前正在执行**的指令。 |
| **PC** | 程序计数器 | 存放**下一条**要执行的指令的地址。==具有自动加1功能==。 |

## 三、 绝对重点：指令执行全过程
> [!abstract] 任何指令的执行，均分为三大阶段：**取指 $\rightarrow$ 分析 $\rightarrow$ 执行**。
> 其中，**取指**和**分析**阶段，对所有指令的操作流程是**完全一致**的！

### 1. 通用阶段：取指令（Fetch）与分析
无论什么指令，CPU第一步永远是去内存把指令拿出来。请死记硬背以下数据流向：
1. **`PC -> MAR`**：PC将下一条指令的地址送给MAR。
2. **`CU -> 主存`**：CU发出读信号。
3. **`主存 -> MDR`**：主存根据MAR的地址，把对应的指令代码取出，放入MDR。
4. **`MDR -> IR`**：MDR中的指令代码送入指令寄存器IR。
5. **`PC自动加1`**：PC更新为下一条指令的地址。
6. **`IR(Opcode) -> CU`**：IR将指令的**操作码**部分送给CU进行分析，CU识别出这是什么指令。

### 2. 分支阶段：执行指令（Execute）
CU分析完操作码后，开始根据指令的不同，指挥数据在 ACC、MQ、X 之间流转，并调用 ALU 运算。
*   **取数指令**：把目标地址送入MAR $\rightarrow$ 读主存送入MDR $\rightarrow$ 送入ACC。
*   **存数指令**：把目标地址送入MAR $\rightarrow$ 数据从ACC送入MDR $\rightarrow$ 写入主存。

> [!danger] 经典必考简答/单选：CPU如何区分取出的二进制是“指令”还是“数据”？
> **答案**：根据**指令执行的阶段（指令周期）**来区分。
> *   在**取指阶段**取出的二进制，一定送往 **IR**，被当作**指令**。
> *   在**执行阶段**取出的二进制，一定送往 **ACC/X等通用运算寄存器**，被当作**数据**。

## 四、 课后回扣：冯·诺依曼计算机特点
（结合本节内容，这几条必须形成本能反应）
1. **指令和数据同等地位**：都在主存里，都用二进制表示。
2. **按地址寻访**：想拿指令还是拿数据，都得通过MAR给地址。
3. **指令的构成**：指令 = **操作码**（干什么事） + **地址码**（去哪找操作数）。
4. **存储程序原理**：运行前，指令和数据必须提前装入主存。（最核心的思想）
