## 一、核心逻辑：一句话总结
**线索化 = 遍历 + 修改指针**。
在遍历（中/先/后）的过程中，利用全局变量（或引用参数）`pre` 指针，记录**当前结点的前驱**，构建“当前结点左空指前驱，前驱右空指当前”的逻辑。

> [!abstract] 核心代码模板 (Visit函数)
> 无论中序、先序、后序，对单个结点的处理逻辑（Visit）是**完全一致**的：
> ```cpp
> void Visit(ThreadNode *q, ThreadNode *&pre) {
>     // 1. 处理当前结点的左孩子（找前驱）
>     if (q->lchild == NULL) {
>         q->lchild = pre;
>         q->ltag = 1;
>     }
>     // 2. 处理前驱结点的右孩子（找后继）
>     // 注意：pre != NULL 才能操作，否则第一个结点会崩
>     if (pre != NULL && pre->rchild == NULL) {
>         pre->rchild = q;
>         pre->rtag = 1;
>     }
>     // 3. 更新pre，为下一个结点做准备
>     pre = q;
> }
> ```

## 二、三种线索化的差异与“防坑”指南

### 1. 中序线索化 (最常规)
*   **遍历顺序**：左 -> 根 -> 右
*   **实现细节**：
    *   `pre` 初始值为 `NULL`。
    *   **收尾处理 (必考点)**：遍历结束后，`pre` 指向最后一个结点。此时 `pre->rchild` 必定为空（否则它不是最后），必须将其处理为线索。
*   **代码结构**：
    ```cpp
    void InThread(ThreadNode *p, ThreadNode *&pre) {
        if (p != NULL) {
            InThread(p->lchild, pre); // 左
            Visit(p, pre);            // 根
            InThread(p->rchild, pre); // 右
        }
    }
    // 主函数调用后：
    if (pre->rchild == NULL) pre->rtag = 1; // 封口
    ```

### 2. 先序线索化 (爱的魔力转圈圈)
*   **遍历顺序**：根 -> 左 -> 右
*   **致命陷阱 (高频考点)**：
    *   在先序中，先访问根（`Visit`），此时若根的左孩子为空，会将其指向 `pre`（即父辈结点）。
    *   紧接着递归进入左子树。若不加判断，会顺着刚连上的线索又回到父辈，导致**死循环**（原地打转）。
*   **解决方案**：
    *   在向左递归前，必须判断 `ltag == 0`（说明是真孩子，不是线索）。
*   **可视化演示**：
    ```mermaid
    graph TD
    A((A)) -->|lchild为空, 线索化指向Pre| Pre[Pre结点]
    A -.->|递归访问左子树| Pre
    Pre -.->|又是A的前驱| A
    style A fill:#f96,stroke:#333,stroke-width:2px
    style Pre fill:#9cf,stroke:#333
    ```
*   **代码修正**：
    ```cpp
    void PreThread(ThreadNode *p, ThreadNode *&pre) {
        if (p != NULL) {
            Visit(p, pre); // 根：此时可能把 p->lchild 改为 pre
            
            // 关键：只有当 lchild 是真孩子时，才往左走！
            if (p->ltag == 0) { 
                PreThread(p->lchild, pre); 
            }
            
            PreThread(p->rchild, pre); // 右
        }
    }
    // 主函数调用后：同样需要处理最后一个结点的 rtag = 1
    ```

### 3. 后序线索化 (安全)
*   **遍历顺序**：左 -> 右 -> 根
*   **特点**：
    *   根节点是最后被访问的。
    *   此时左右子树都已经处理完毕，不会出现先序那种“刚连上线索又钻回去”的问题。
    *   **无需**像先序那样判断 `ltag`。
*   **注意**：主函数结束后，依然要处理最后一个结点的 `rtag = 1`。

## 三、代码实现细节 (408标准)

### 1. 结构体定义
```cpp
typedef struct ThreadNode {
    ElemType data;
    struct ThreadNode *lchild, *rchild;
    int ltag, rtag; // 0:孩子, 1:线索
} ThreadNode, *ThreadTree;
```

### 2. 全局变量 vs 引用参数
*   **推荐写法 (王道风格)**：使用 `ThreadNode *&pre` 作为函数参数。
    *   **原因**：引用传递相当于全局变量，但在考场手写代码时，封装性更好，不易漏写初始状态。
*   **初始化**：
    ```cpp
    ThreadNode *pre = NULL; // 必须初始化为NULL
    if (T != NULL) {
        InThread(T, pre); // 或 PreThread, PostThread
        // 处理最后一个结点
        if (pre->rchild == NULL) pre->rtag = 1;
    }
    ```

## 四、极简记忆表 (背诵用)

| 维度 | 中序线索化 | 先序线索化 | 后序线索化 |
| :--- | :--- | :--- | :--- |
| **Visit位置** | 中间 (左**根**右) | 最前 (**根**左右) | 最后 (左右**根**) |
| **死循环风险** | 无 | **有 (需判ltag==0)** | 无 |
| **最后结点处理**| 需要 (`rtag=1`) | 需要 (`rtag=1`) | 需要 (`rtag=1`) |
| **难度** | ⭐ | ⭐⭐⭐ (转圈问题) | ⭐ |

> [!fail] 常见丢分点
> 1.  忘记初始化 `pre = NULL`。
> 2.  **先序遍历**递归左子树时，忘记写 `if (p->ltag == 0)`，导致死循环。
> 3.  遍历结束后，忘记单独处理最后一个结点的右指针（封口）。
> 4.  混淆 `ltag=0` (孩子) 和 `ltag=1` (线索) 的定义。
