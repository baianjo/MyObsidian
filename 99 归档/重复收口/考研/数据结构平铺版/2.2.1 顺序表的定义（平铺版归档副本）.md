### 一、 核心定义与存储结构
**定义**：用**顺序存储**方式实现的线性表。
**核心特征**：
1.  **逻辑上相邻**的数据元素，在**物理存储位置上也相邻**。
2.  数据类型相同（占用的存储空间大小相等）。

**地址计算公式**（考点：随机访问的基础）：
$$Loc(L_i) = Loc(L_1) + (i-1) \times \text{sizeof(ElemType)}$$
> **注意**：线性表位序从1开始，数组下标从0开始。

### 二、 两种实现方式（代码层面）

#### 1. 静态分配（Static Allocation）
*   **机制**：使用固定长度的数组 `data[MaxSize]`。
*   **特点**：容量一旦确定**不可改变**。如果存满，会溢出，这就叫"放弃治疗"。
*   **代码模板**：
```c
#define MaxSize 10      // 定义最大长度
typedef struct {
    ElemType data[MaxSize]; // 静态数组
    int length;             // 当前长度
} SqList;
```
*   **初始化注意**：必须将 `length` 设为 0。虽然后面的脏数据（Dirty Data）通常访问不到，但严谨的初始化建议将 `data` 数组也清零。

#### 2. 动态分配（Dynamic Allocation）
*   **机制**：使用指针 `*data` 指向 `malloc` 分配的堆区内存。
*   **特点**：容量**可动态增加**，但需要付出时间代价（申请新空间 -> 复制数据 -> 释放旧空间）。
*   **代码模板**：
```c
#define InitSize 10     // 默认初始长度
typedef struct {
    ElemType *data;     // 指示动态分配数组的指针
    int MaxSize;        // 顺序表的最大容量
    int length;         // 顺序表的当前长度
} SeqList;

// 初始化
L.data = (ElemType *)malloc(sizeof(ElemType) * InitSize);

动态扩容逻辑（核心步骤）
1. p = L.data;
2. L.data = (ElemType *)malloc(sizeof(ElemType) * (L.MaxSize + len));
3. for循环复制数据
4. L.MaxSize = L.MaxSize + len;
5. free(p); // 释放旧空间
```
*   **关键函数**：`malloc`（申请）、`free`（释放）。
*   **C语言细节**：`malloc` 返回的是 `void*`，必须**强制类型转换**为对应数据类型的指针。

### 三、 顺序表的四大特性（背诵/选择题考点）

| 特性 | 描述 | 考研评价 |
| :--- | :--- | :--- |
| **1. 随机访问** | **最重要的特性**。可以在 $O(1)$ 时间内找到第 $i$ 个元素。 | **得分点**，利用首地址+偏移量直接计算。 |
| **2. 存储密度高** | 只需要存储数据本身，不需要像链表那样存指针。 | 优点。 |
| **3. 扩展容量不便** | 静态无法扩容；动态扩容需要移动大量数据，时间代价高。 | 缺点。 |
| **4. 插入删除不便** | 需要移动大量元素以保持物理连续性。 | 缺点（下节重点）。 |

### 四、 避坑指南 & 拿分策略

1.  **位序 vs 下标**：
    *   题目说“第 $i$ 个元素”，通常指位序，对应数组下标 `[i-1]`。
    *   代码实现时一定要分清逻辑序号（1开始）和物理序号（0开始）。

2.  **脏数据（Dirty Data）**：
    *   内存中遗留的旧数据。
    *   **策略**：初始化时务必 `length = 0`。访问数据时，严格限制在 `0` 到 `length-1` 范围内，不要越界访问。

3.  **动态扩容的代价**：
    *   虽然叫“动态”，但它不是原地扩容，而是**“搬家”**。
    *   时间复杂度高，不要滥用。

4.  **sizeof 使用**：
    *   `sizeof(int)` 通常是 4 字节。
    *   `malloc` 参数是 **字节数**，不是元素个数。公式：`sizeof(ElemType) * 个数`。
