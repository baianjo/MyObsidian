> [!abstract] **核心考点速览**
> 1.  **头结点**：默认带头结点，代码统一且不易错。
> 2.  **核心操作**：插入（防断链）、删除（防内存泄漏）、建表（头插逆序、尾插正序）。
> 3.  **高分技巧**：$O(1)$ 时间复杂度实现前插和删除指定结点（偷天换日法）。
> 4.  **边界条件**：表头、表尾、空表判断（985上岸必须严谨）。

## 1. 结构定义与初始化 (Standard)
> **功利化建议**：考试手写代码默认**带头结点**，除非题目强制要求不带。带头结点可以处理第1个位置和其他位置逻辑统一，无需特殊判断。

```c
typedef struct LNode {
    ElemType data;
    struct LNode *next;
} LNode, *LinkList;

// 初始化（带头结点）
bool InitList(LinkList &L) {
    L = (LNode*) malloc(sizeof(LNode)); // 分配头结点
    if (L == NULL) return false;        // 内存不足
    L->next = NULL;                     // !重要：头结点指针域置空，防脏数据
    return true;
}
```

## 2. 插入操作 (Insertion)

### 2.1 按位序插入 (Insert at i)
**目标**：在第 $i$ 个位置插入元素 $e$。
**思路**：找到第 $i-1$ 个结点，执行后插。
**复杂度**：$O(n)$ (平均/最坏)，$O(1)$ (最好，i=1)。

```c
// 在第 i 个位置插入 e（带头结点）
bool ListInsert(LinkList &L, int i, ElemType e) {
    if (i < 1) return false;
    LNode *p = L; // 指向头结点（第0个）
    int j = 0;
    
    // 寻找第 i-1 个结点
    while (p != NULL && j < i - 1) {
        p = p->next;
        j++;
    }
    
    // 边界检查：p不存在（i值过大）
    if (p == NULL) return false;
    
    // 核心代码：后插操作
    LNode *s = (LNode*) malloc(sizeof(LNode));
    s->data = e;
    s->next = p->next; // 1. 先连后继
    p->next = s;       // 2. 后断前驱（顺序不可颠倒！）
    return true;
}
```

> [!danger] **考点警示：断链风险**
> `s->next = p->next;` 和 `p->next = s;` **绝对不能颠倒**！
> 若颠倒，`p->next` 先被修改，原链表后续结点全部丢失。

### 2.2 指定结点后插 (Insert Next)
**场景**：已知结点 $p$，在 $p$ 之后插入 $s$。
**复杂度**：$O(1)$。
```c
s->next = p->next;
p->next = s;
```

### 2.3 指定结点前插 (Insert Prior) —— **高频骚操作**
**场景**：已知结点 $p$，在 $p$ 之前插入 $s$。
**常规法**：从头遍历找前驱，$O(n)$。
**技巧法（偷天换日）**：插在 $p$ 之后，交换数据。$O(1)$。

```c
// 在 p 结点之前插入 e
bool InsertPriorNode(LNode *p, ElemType e) {
    if (p == NULL) return false;
    LNode *s = (LNode*) malloc(sizeof(LNode));
    if (s == NULL) return false; // 内存分配失败
    
    // 1. 物理上插在 p 之后
    s->next = p->next;
    p->next = s;
    
    // 2. 逻辑上交换数据（把 p 的旧数据挤到 s 去，p 存新数据）
    s->data = p->data;
    p->data = e;
    
    return true;
}
```

## 3. 删除操作 (Deletion)

### 3.1 按位序删除 (Delete at i)
**目标**：删除第 $i$ 个结点。
**思路**：找到第 $i-1$ 个结点 $p$，修改指针跳过第 $i$ 个结点 $q$，并 `free(q)`。
**复杂度**：$O(n)$。

```c
bool ListDelete(LinkList &L, int i, ElemType &e) {
    if (i < 1) return false;
    LNode *p = L;
    int j = 0;
    
    // 找第 i-1 个结点
    while (p != NULL && j < i - 1) {
        p = p->next;
        j++;
    }
    
    // p不存在 或 p是最后一个结点（导致要删的第i个结点不存在）
    if (p == NULL || p->next == NULL) return false;
    
    LNode *q = p->next; // 令 q 指向被删结点
    e = q->data;        // 保存数据
    p->next = q->next;  // 断链
    free(q);            // 释放内存
    return true;
}
```

### 3.2 删除指定结点 (Delete Node p) —— **考点技巧**
**场景**：只给结点指针 $p$，不给头指针。
**技巧法（偷天换日）**：将 $p$ 后继结点的数据复制给 $p$，然后删除后继结点。$O(1)$。

```c
// 删除指定结点 p
bool DeleteNode(LNode *p) {
    if (p == NULL) return false;
    
    // 核心缺陷/边界条件：如果 p 是最后一个结点，此法无效！
    // 必须从头遍历找到前驱，时间复杂度退化为 O(n)
    if (p->next == NULL) {
        // 考试时若只能写这段，需注释说明：此时需要传入头指针处理
        return false; 
    }
    
    LNode *q = p->next; // q 是 p 的后继
    p->data = q->data;  // 数据覆盖：后继的值给 p
    p->next = q->next;  // 链表跳过 q
    free(q);
    return true;
}
```

## 4. 查找操作 (Search)

*   **按位查找 (GetElem)**：$O(n)$。`p` 从头往后走 `i` 步。
*   **按值查找 (LocateElem)**：$O(n)$。`while(p!=NULL && p->data != e)`。
    *   *注意*：结构体比较不能直接用 `==`，但在408手写代码中，若元素为 `int` 等基本类型直接对比；若为复杂结构体，默认假设已重载 `==` 或逻辑上可行。

## 5. 单链表的建立 (Creation)

### 5.1 尾插法 (Tail Insert) —— **正向建表**
**特点**：顺序一致（输入 1,2,3 -> 链表 1,2,3）。
**必须**：维护一个尾指针 `r`，始终指向最后一个结点。
**复杂度**：$O(n)$。

```c
LinkList List_TailInsert(LinkList &L) {
    // 1. 初始化
    L = (LNode*) malloc(sizeof(LNode));
    L->next = NULL; // 初始为空表
    LNode *s, *r = L; // r 为尾指针
    
    int x;
    scanf("%d", &x);
    while (x != 9999) { // 9999为结束标记
        s = (LNode*) malloc(sizeof(LNode));
        s->data = x;
        r->next = s; // 挂在尾部
        r = s;       // 更新尾指针 r 指向新的尾部
        scanf("%d", &x);
    }
    r->next = NULL; // !重要：收尾，尾结点指针置空
    return L;
}
```

### 5.2 头插法 (Head Insert) —— **逆向建表/链表逆置**
**特点**：顺序相反（输入 1,2,3 -> 链表 3,2,1）。
**应用**：**题目要求“链表逆置”时，直接原地使用头插法**。
**复杂度**：$O(n)$。

```c
LinkList List_HeadInsert(LinkList &L) {
    L = (LNode*) malloc(sizeof(LNode));
    L->next = NULL; // !重要：必须初始化为NULL，否则最后尾部是野指针
    LNode *s;
    int x;
    scanf("%d", &x);
    while (x != 9999) {
        s = (LNode*) malloc(sizeof(LNode));
        s->data = x;
        s->next = L->next; // 新结点指向原首元结点
        L->next = s;       // 头结点指向新结点
        scanf("%d", &x);
    }
    return L;
}
```

## 6. 避坑指南 & 拿分逻辑

1.  **带头结点 vs 不带头结点**：
    *   **带头结点**：头指针 `L` 指向头结点，`L->next` 才是第1个数据。代码统一。
    *   **不带头结点**：`L` 指向第1个数据。插入/删除第1个元素时，需要修改 `L` 本身。
    *   **结论**：写代码一定要审题。若未规定，**默认带头结点**写起来最快且不出错。

2.  **鲁棒性 (Robustness)**：
    *   函数入口检查：`if (L == NULL)`。
    *   位置合法性检查：`i < 1`。
    *   空指针访问检查：`p->next` 之前确认 `p` 存在。
    *   内存分配检查：`malloc` 后检查是否为 `NULL`（手写非必须，但加分）。

3.  **时间复杂度总结**：
    *   查找/按位序插入/按位序删除：$O(n)$。
    *   已知结点后插/前插/删除：$O(1)$。
    *   求表长：$O(n)$。
    *   建表：$O(n)$。
