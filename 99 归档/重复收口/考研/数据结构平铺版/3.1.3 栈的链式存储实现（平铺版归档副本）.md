> [!abstract] **核心考点摘要**
> 1.  **定义**：**链栈 (Link Stack)** = 单链表 + 操作限制（只能在**表头**进行插入和删除）。
> 2.  **物理结构**：为了操作方便（$O(1)$），**链表的头部**作为**栈顶**。
> 3.  **核心操作映射**：
>     *   **进栈 (Push)** $\Leftrightarrow$ 单链表的**头插法**。
>     *   **出栈 (Pop)** $\Leftrightarrow$ 单链表的**对头结点的后继进行删除**。
> 4.  **优势**：不存在“栈满/上溢”的情况（除非内存耗尽），便于动态扩充。

---

### 一、 存储结构定义

既然目标是985，代码手写必须规范。虽然课件中提到课本推荐“不带头结点”，但在408及自主命题中，**带头结点**的统一性更好，建议两者都掌握，以防万一。

```cpp
typedef struct LinkNode {
    ElemType data;
    struct LinkNode *next;
} LinkNode, *LinkStack;
```

### 二、 核心操作代码模板 (背诵级)

> [!WARNING] **防丢分警示**
> 讲师为了让你练习故意没给代码，但考试通过率靠的是**标准答案**。以下是**带头结点**的标准写法（最通用）。

#### 1. 初始化
```cpp
bool InitStack(LinkStack &S) {
    S = (LinkNode *)malloc(sizeof(LinkNode));
    if (S == NULL) return false; // 内存分配失败
    S->next = NULL; // 头结点指针域置空
    return true;
}
```

#### 2. 判空 (IsEmpty)
*   **带头结点**：`return S->next == NULL;`
*   **不带头结点**：`return S == NULL;`

#### 3. 进栈 (Push) -> 头插法
**时间复杂度**：$O(1)$
```cpp
bool Push(LinkStack &S, ElemType x) {
    LinkNode *p = (LinkNode *)malloc(sizeof(LinkNode));
    if (p == NULL) return false; // 内存满
    p->data = x;
    
    // 核心逻辑：头插法
    p->next = S->next;
    S->next = p; 
    
    return true;
}
```

#### 4. 出栈 (Pop) -> 删除首元结点
**时间复杂度**：$O(1)$
```cpp
bool Pop(LinkStack &S, ElemType &x) {
    if (S->next == NULL) return false; // 栈空
    
    LinkNode *p = S->next; // p指向栈顶元素（头结点的后继）
    x = p->data;           // 获取数据
    
    // 核心逻辑：断链
    S->next = p->next;     // 头结点指向栈顶的下一个
    
    free(p);               // 释放内存
    return true;
}
```

### 三、 考点对比：带头结点 vs 不带头结点

对于985考生，必须对这两种实现的细微差别如数家珍，选择题常坑人。

| 比较维度 | **带头结点 (推荐/通用)** | **不带头结点 (课本提及)** |
| :--- | :--- | :--- |
| **头指针 S** | 指向一个不存数据的Dummy Node | 直接指向栈顶第一个实际元素 |
| **判空条件** | `S->next == NULL` | `S == NULL` |
| **进栈逻辑** | 新结点插入 `S` 之后 | 新结点成为新的 `S`，需修改头指针 |
| **操作统一性** | 空表和非空表操作逻辑一致，不易出错 | 空表插入需特殊处理 |

> [!tip] **应试策略**
> 1.  **代码题**：如果题目未明确规定，默认写**带头结点**的版本，逻辑更清晰，代码更少（不用特判空栈插入）。
> 2.  **选择题**：注意看题干描述，如果提到“栈顶指针指向第一个元素”，通常暗示**不带头结点**。

### 四、 避坑指南 (易错点)

1.  **进出端选择**：链栈的栈顶**必须是链头**。
    *   *原因*：若以链尾为栈顶，入栈（尾插）需遍历到尾部（除非带尾指针），出栈（删除尾部）需找到倒数第二个结点（必须遍历），时间复杂度为 $O(n)$，不符合栈 $O(1)$ 的要求。
2.  **满栈问题**：链栈理论上没有“栈满”限制（除非内存不足），这点常与**顺序栈**（固定大小，有`StackOverflow`）做对比考察。
3.  **指针断链**：在`Pop`操作中，务必先保留要删除结点的`next`指针或直接接上链，再`free`，切忌顺序颠倒导致断链。
