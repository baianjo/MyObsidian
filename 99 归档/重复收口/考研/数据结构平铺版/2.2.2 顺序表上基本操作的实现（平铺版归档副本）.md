> [!summary] 核心考点速记
> **顺序表 (Sequential List)** 最核心特性是 **随机存取 (Random Access)**，即按位查找 $O(1)$。
> 但 **插入/删除** 需要移动大量元素，平均时间复杂度为 $O(n)$。
>
> 🚩 **高分警示**：
> 1. **位序 vs 下标**：题目说“第 $i$ 个位置”，代码对应下标 `i-1`。
> 2. **健壮性**：必须检查 $i$ 的合法性及表是否满/空，这是**大题不丢分**的关键。
> 3. **引用符号 `&`**：修改原数据必须加引用（C++语法），否则操作的是副本。

## 一、 基本操作复杂度总表

| 操作类型     | 操作描述               | 最好情况        | 最坏情况        | 平均情况       | 关注点         |
| :------- | :----------------- | :---------- | :---------- | :--------- | :---------- |
| **插入**   | `Insert(L, i, e)`  | $O(1)$ (表尾) | $O(n)$ (表头) | **$O(n)$** | 元素后移，从后往前遍历 |
| **删除**   | `Delete(L, i, &e)` | $O(1)$ (表尾) | $O(n)$ (表头) | **$O(n)$** | 元素前移，从前往后遍历 |
| **按位查找** | `GetElem(L, i)`    | $O(1)$      | $O(1)$      | **$O(1)$** | 直接通过数组下标访问  |
| **按值查找** | `LocateElem(L, e)` | $O(1)$ (表头) | $O(n)$ (表尾) | **$O(n)$** | 需注意结构体比较问题  |

## 二、 插入操作 (ListInsert)

**逻辑**：在第 $i$ 个位置（下标 `i-1`）插入元素 $e$。需要将下标 `i-1` 及之后的元素**依次向后移动**。

> [!danger] 避坑指南：移动方向
> 必须 **从后往前** 移动元素，否则会覆盖数据。
> 循环范围：`j` 从 `length` 到 `i`。

```cpp
bool ListInsert(SqList &L, int i, int e) {
    // 1. 判断 i 的合法性 (位序范围：1 ~ length+1)
    if (i < 1 || i > L.length + 1) return false;
    
    // 2. 判断存储空间是否已满
    if (L.length >= MaxSize) return false;

    // 3. 核心逻辑：将第 i 个元素及之后的元素后移
    // 注意：j 代表的是位序，对应下标 j-1
    for (int j = L.length; j >= i; j--) {
        L.data[j] = L.data[j-1]; // data[j] 是空位，把前一个移过来
    }

    // 4. 插入新元素
    L.data[i-1] = e;
    
    // 5. 长度 +1
    L.length++;
    
    return true;
}
```

## 三、 删除操作 (ListDelete)

**逻辑**：删除第 $i$ 个位置（下标 `i-1`）的元素，并用 `e` 返回其值。需要将下标 `i` 及之后的元素**依次向前移动**。

> [!tip] 关键点：引用参数 `&`
> 参数 `int &e` 表示**引用**。在函数内修改 `e`，主函数中的对应变量也会改变。
> 若不加 `&`，函数操作的只是副本，主函数拿不到被删除的值。

```cpp
bool ListDelete(SqList &L, int i, int &e) {
    // 1. 判断 i 的合法性 (位序范围：1 ~ length)
    // 注意：删除操作 i 不能等于 length+1，因为那里没数据
    if (i < 1 || i > L.length) return false;

    // 2. 取出被删除元素
    e = L.data[i-1];

    // 3. 核心逻辑：将第 i 个位置后的元素前移
    // 移动方向：从前往后
    for (int j = i; j < L.length; j++) {
        L.data[j-1] = L.data[j]; // 把后一个元素填补到前一个位置
    }

    // 4. 长度 -1
    L.length--;
    
    return true;
}
```

## 四、 查找操作

### 1. 按位查找 (GetElem)
由于顺序表在物理内存中是**连续存放**的，且每个元素大小相同，已知首地址即可直接计算目标地址。
*   **代码核心**：`return L.data[i-1];`
*   **跨考注意**：C语言指针访问 `data[i]` 时，系统会自动根据数据类型大小（如 int 占 4B）计算偏移量，无需手动乘字节数。

### 2. 按值查找 (LocateElem)
**逻辑**：从头到尾遍历，对比值是否相等。
*   **代码核心**：
    ```cpp
    for(int i=0; i<L.length; i++){
        if(L.data[i] == e) return i+1; // 返回位序
    }
    return 0; // 未找到
    ```
*   **高分陷阱（结构体比较）**：
    *   如果是基本类型（`int`, `float`），直接用 `==`。
    *   如果是 **struct** 类型，C语言**不能**直接用 `==` 比较两个结构体。
    *   **考试策略**：
        *   若考题科目为“数据结构”，通常允许写 `element == e`（逻辑层）。
        *   若考题科目偏“C语言程序设计”，必须逐个比较成员变量（如 `a.id == b.id`）。

## 五、 易错点总结 (Self-Check)

- [ ] **位序 vs 下标**：代码中涉及到数组访问，一定要把题目给的位序 $i$ 转换为下标 $i-1$。
- [ ] **循环边界**：
    - 插入：`j >= i` （要把第 $i$ 个位置腾出来）
    - 删除：`j < L.length` （要把最后一个元素移过来）
- [ ] **返回值**：布尔类型 `bool` 用于反馈操作成功/失败，这是**代码健壮性**的体现（阅卷老师看重的点）。
- [ ] **平均时间复杂度推导**：
    - 插入/删除的平均移动次数约为 $\frac{n}{2}$。
    - 概率 $p = \frac{1}{n+1}$ (插入) 或 $\frac{1}{n}$ (删除)，结合等差数列求和公式推导。
