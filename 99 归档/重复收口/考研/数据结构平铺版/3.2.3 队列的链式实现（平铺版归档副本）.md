# 链队列 (Linked Queue)

> [!SUMMARY] **核心结论 (功利化记忆)**
> *   **本质**：受限的单链表（只在表尾插入，只在表头删除）。
> *   **结构**：需要两个指针 `front` (队头) 和 `rear` (队尾)。
> *   **考点**：**带头结点** vs **不带头结点** 的代码差异；**出队时若队列变空**，需特殊处理 `rear` 指针。
> *   **优势**：相比顺序队列，**不存在“假溢出”**，扩容方便。

---

## 1. 存储结构定义

为了实现 $O(1)$ 的入队（尾插），除了头指针，必须维护一个**尾指针**。

```cpp
// 1. 链表节点定义
typedef struct LinkNode {
    ElemType data;
    struct LinkNode *next;
} LinkNode;

// 2. 链队列定义 (包裹两个指针)
typedef struct {
    LinkNode *front, *rear; // 队头、队尾指针
} LinkQueue;
```

---

## 2. 核心操作对比 (带头结点 vs 不带头结点)

**高分策略**：考试默认写**带头结点**版本（代码更统一），除非题目强制要求不带。

### 2.1 初始化 & 判空

| 操作 | **带头结点 (推荐)** | 不带头结点 (易错) |
| :--- | :--- | :--- |
| **图示** | `[Front/Rear] -> [HeadNode] -> NULL` | `Front = NULL`, `Rear = NULL` |
| **初始化** | `Q.front = Q.rear = (LinkNode*)malloc(...)`<br>`Q.front->next = NULL;` | `Q.front = NULL;`<br>`Q.rear = NULL;` |
| **判空** | `Q.front == Q.rear` | `Q.front == NULL` |

---

### 2.2 入队 (Enqueue) - $O(1)$
*逻辑：尾部插入新节点 `s`，更新 `rear`。*

*   **带头结点**：无需特殊判断。
    ```cpp
    s->data = x; s->next = NULL;
    Q.rear->next = s;  // 挂在尾部
    Q.rear = s;        // 更新尾指针
    ```
*   **不带头结点**：**必考坑点！** 插入第一个元素时，`front` 和 `rear` 都要修改。
    ```cpp
    s->data = x; s->next = NULL;
    if (Q.front == NULL) { // 插入的是第一个元素
        Q.front = s;
        Q.rear = s;
    } else {
        Q.rear->next = s;
        Q.rear = s;
    }
    ```

---

### 2.3 出队 (Dequeue) - $O(1)$
*逻辑：头部删除节点，释放内存。*

> [!DANGER] **丢分高危区：当队列中只剩一个元素时**
> 如果删除了最后一个元素，必须将 `rear` 指针修复（指回 `head` 或 `NULL`），否则 `rear` 会变成野指针。

#### **方案 A：带头结点 (标准写法)**
```cpp
bool DeQueue(LinkQueue &Q, ElemType &x) {
    if (Q.front == Q.rear) return false; // 空队
    
    LinkNode *p = Q.front->next; // p指向待删节点(首元节点)
    x = p->data;
    
    Q.front->next = p->next;     // 修改头结点的next
    
    // 【核心满分点】若原队列只有一个结点，删后变空，rear需指回front
    if (Q.rear == p) {
        Q.rear = Q.front;
    }
    
    free(p);
    return true;
}
```

#### **方案 B：不带头结点**
```cpp
bool DeQueue(LinkQueue &Q, ElemType &x) {
    if (Q.front == NULL) return false; // 空队
    
    LinkNode *p = Q.front; // p指向待删节点
    x = p->data;
    
    Q.front = p->next;     // front 后移
    
    // 【核心满分点】若删的是最后一个结点，删后变空
    if (Q.rear == p) {
        Q.front = NULL;
        Q.rear = NULL; // 必须重置为NULL
    }
    
    free(p);
    return true;
}
```

## 3. 逻辑可视化 (Obsidian 视图)

**状态：带头结点队列**

1.  **空队列**：
    ```mermaid
    graph LR
    F[Front] --> H[HeadNode]
    R[Rear] --> H
    H --> NULL
    ```

2.  **一般状态 (入队 A, B)**：
    ```mermaid
    graph LR
    F[Front] --> H[HeadNode]
    R[Rear] --> B
    H --> A --> B --> NULL
    ```

3.  **出队特殊情况 (只剩 B，删除 B)**：
    *   删除前：`Front->Head->B`, `Rear->B`
    *   删除后：`Head->NULL`
    *   **关键动作**：`Rear` 必须强行拉回指向 `Head`，否则 `Rear` 指向已 free 的 `B`。

## 4. 扩展考点 (防坑)

1.  **求队长**：
    *   标准链队列求长度需要从头遍历，时间复杂度 **$O(n)$**。
    *   *优化技巧*：如果在结构体中增加 `int length;` 成员，增删时维护该变量，则求队长为 $O(1)$。代码题可根据需求灵活添加。
2.  **空间分配**：
    *   链队列适合数据量波动大、无法预估长度的场景。
    *   不存在“队满”的情况（除非内存耗尽），故一般不需要 `Full` 判断。
3.  **双端队列 (Deque)**：
    *   如果需要尾删 (Delete at Rear)，单链表实现的队列需要遍历找前驱，复杂度 $O(n)$。若要 $O(1)$ 尾删，需使用**双向链表**。
