**Q5. 排序（秒杀题）**
请按照时间复杂度**从低到高**排序以下几项（直接写序号）：
1. $O(n^2)$
2. $O(n!)$
3. $O(n \log n)$
4. $O(n)$
5. $O(2^n)$
6. $O(\sqrt{n})$

**排序结果：__________**

---

### ✅ 答案与诊断

**Answer 1:** $O(\log_2 n)$ 或 $O(\log n)$
*   **解析**：$i$ 每次乘2，增长速度是 $2^k = n \Rightarrow k = \log_2 n$。这是STT里讲了半天的核心点。
*   **130+要求**：必须立刻反应出是对数级。

**Answer 2:** $O(n^2)$
*   **解析**：这是等差数列求和。执行次数为 $1 + 2 + 3 + ... + n = \frac{n(n+1)}{2}$。最高阶项是 $n^2$。
*   **130+要求**：不能死背“双层循环就是平方”，要看到内层循环次数取决于外层 $i$，但因为求和公式最高次是平方，所以结果是平方。

**Answer 3:** $O(\sqrt{n})$
*   **解析**：这是很多考生的盲区。
    *   $i$ 的变化：$1, 2, 3, ..., k$
    *   $sum$ 的变化：$1, 1+2, 1+2+3, ..., \frac{k(k+1)}{2}$
    *   循环结束条件：$sum \ge n$，即 $\frac{k(k+1)}{2} \approx n$
    *   所以 $k^2 \approx n \Rightarrow k \approx \sqrt{n}$。
*   **130+要求**：如果这题你答成了 $O(n)$ 或 $O(\log n)$，说明你的基础有漏洞，这课虽然浅，但你必须去找这种“条件非线性”的题练一下。

**Answer 4:** $O(n)$
*   **解析**：这是一个简单的线性递归，递归深度为 $n$，每次函数内部操作是常数级 $O(1)$。总复杂度 = $n \times 1 = O(n)$。

**Answer 5:** $6 < 4 < 3 < 1 < 5 < 2$
*   **即**：$O(\sqrt{n}) < O(n) < O(n \log n) < O(n^2) < O(2^n) < O(n!)$
*   **解析**：常对幂指阶。注意 $\sqrt{n}$ (也就是 $n^{0.5}$) 小于 $n$。


---
#### 1. 核心结论（必须背诵）
*   **大O表示法只看最高阶**：忽略低阶项，忽略常数系数。
    *   $3n \rightarrow O(n)$
    *   $n^2 + n \rightarrow O(n^2)$
*   **复杂度排序口诀**（STT里提到的“常对幂指阶”）：
    $$O(1) < O(\log_2 n) < O(n) < O(n \log_2 n) < O(n^2) < O(n^3) < O(2^n) < O(n!)$$
    *   **考试技巧**：当 $n$ 足够大时，右边的永远比左边的“大/慢”。

#### 2. 计算规则（做题用）
*   **加法规则**：多项相加，只保留最高阶。
*   **乘法规则**：嵌套循环，复杂度相乘。
    *   外层 $n$ 次，内层 $n$ 次 $\rightarrow O(n^2)$。

#### 3. 实战判定技巧（重点）
*   **只看最深层循环**：不管代码几千行，只盯着循环次数最多的那一句核心语句。
*   **经典模型（必考）**：
    *   **普通循环**：`for(i=0; i<n; i++)` $\rightarrow O(n)$
    *   **双层嵌套**：$\rightarrow O(n^2)$
    *   **倍增循环（难点）**：
        ```c
        while(i < n) {
            i = i * 2; // 每次翻倍
        }
        ```
        *   **结论**：$2^x = n \Rightarrow x = \log_2 n$，复杂度为 **$O(\log_2 n)$**。
        *   同理，如果是 `i = i * 3` 也是对数级。

#### 4. 最好/最坏/平均复杂度
*   **定义**：有些算法（如查找）的快慢取决于数据原本的顺序。
    *   运气最好（第一个就是）：$O(1)$
    *   运气最差（最后一个才是）：$O(n)$
    *   平均情况：$O(n)$
*   **408 侧重点**：一般情况下，我们只关注**最坏情况**和**平均情况**。算法的优劣通常由最坏情况决定。
